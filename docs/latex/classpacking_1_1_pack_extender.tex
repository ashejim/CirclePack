\doxysubsection{Detailed Description}
Abstract base for optional "{}extender"{} modules that add specialized commands and data to a circle packing. 

A \doxylink{classpacking_1_1_pack_extender}{Pack\+Extender} is a plug-\/in mechanism that attaches to a \doxylink{classpacking_1_1_pack_data}{Pack\+Data} and provides additional commands, data storage, and GUI elements for a specific mathematical experiment or application (e.\+g. \doxylink{classschwarz_work_1_1_schwarzian}{Schwarzian} derivatives, conformal tiling, Beltrami flows). Extenders are loaded dynamically and register their commands with the packing\textquotesingle{}s command parser.

\end{DoxySeeAlso}

An abstract class for optional structures/\+methods extending \textquotesingle{}\doxylink{classall_mains_1_1_circle_pack}{Circle\+Pack}\textquotesingle{}. Examples are (or will be):\+ Brooks packings, dessin d\textquotesingle{}enfants, complex analysis, curvature flow, etc. Some are called by abbreviations:\+ several are hard coded in \textquotesingle{}\doxylink{classinput_1_1_command_str_parser_ad2c354ce742f567dba1b7cd963fb0323}{Command\+Str\+Parser.\+jexecute}\textquotesingle{}, but \textquotesingle{}Pack\+Extender\+Loader\textquotesingle{} should allow the user to load self-\/written \textquotesingle{}Pack\+Extenders\textquotesingle{} (once they have been compiled to \texorpdfstring{$\ast$}{*}.class)

The idea is that when a packing is created for these special purposes, then the packing points to the \doxylink{classpacking_1_1_pack_extender}{Pack\+Extender} and vice verse. Commands can be constructed that apply to a packing iff it has the appropriate extender abbreviation. \begin{DoxyAuthor}{Author}
kens
