\doxysubsection{Detailed Description}
A point on the Riemann sphere with both "{}real"{} and "{}apparent"{} coordinates. 

Extends {\ttfamily \doxylink{classcomplex_1_1_complex}{Complex}} to represent a point on the unit sphere that carries two coordinate systems simultaneously:\+
\begin{DoxyItemize}
\item {\bfseries{Real}} spherical coordinates (theta, phi) — the mathematical position on the sphere.
\item {\bfseries{Apparent}} coordinates — the screen-\/projected position after applying a {\ttfamily \doxylink{classmath_1_1_matrix3_d}{Matrix3D}} orthogonal transformation that encodes the current viewing orientation.
\end{DoxyItemize}

The inherited (x, y) fields hold the {\itshape apparent} (theta, phi) after the viewing transformation is applied in the constructor.

\begin{DoxyNote}{Note}
The apparent-\/sphere coordinates are ephemeral; they must be recomputed whenever the viewing orientation changes. The {\ttfamily \doxylink{}{on\+Front}} flag indicates whether the point is on the hemisphere facing the viewer.
\end{DoxyNote}

Apply an orthogonal transformation to convert between real and apparent spherical coordinates. 

When {\ttfamily to\+A == true}, the transformation maps a real (theta, phi) point to its apparent-\/sphere (theta, phi) using the orthogonal matrix {\ttfamily trans}. When {\ttfamily to\+A == false}, the inverse (transpose) is applied, mapping apparent → real coordinates.

The algorithm proceeds in three steps:\+
\begin{DoxyEnumerate}
\item Convert (theta, phi) → Cartesian (x, y, z) on the unit sphere.
\item Multiply by {\ttfamily trans} (or its transpose).
\item Convert the resulting (x, y, z) back to (theta, phi).
\end{DoxyEnumerate}

\begin{DoxyNote}{Note}
This method may no longer be called directly; see {\ttfamily to\+Apparent\+Sph} and {\ttfamily to\+Real\+Sph} in \doxylink{_sph_view_8java}{Sph\+View.\+java}.
\end{DoxyNote}
