\doxysubsection{Detailed Description}
Pack\+Extender for exploring and computing discrete \doxylink{classschwarz_work_1_1_schwarzian}{Schwarzian} derivative maps. 

This is code for exploring discrete \doxylink{classschwarz_work_1_1_schwarzian}{Schwarzian} derivatives. One goal is to handle packings on the sphere, another is to study projective structures on more general Riemann surfaces. Motivated by Gerald Orick\textquotesingle{}s thesis, I am pursuing a slightly modified version of his "{}directed Mobius edge derivative"{} along with an "{}intrinsic"{} schwarzian associated with maps from a standardized "{}base equilateral"{}.

DISCRETE SCHWARZIAN DERIVATIVE:\+ The concern is with maps between tangency circle packings sharing the same combinatorics. A circle packing "{}patch"{} refers to two faces sharing an edge e, thus sharing two neighboring circles Cv, Cw. Given map phi:\+P -\/-\/\texorpdfstring{$>$}{>} P\textquotesingle{} between circle packings, for each face f and f\textquotesingle{}=phi(f), M(f) is the "{}face Mobius map"{} of f onto f\textquotesingle{}. This identifies corresponding tangency points and hence maps one triple of circles to the other. (These face mappings were pieced together, along with extensions to circle interiors, by He/\+\+Schramm to define point mappings between circle packing carriers and was used by them to prove convergence results.)

\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*} CONVENTION:\+ Given directed edge \{v,w\} shared by faces f and g, we reverse Orick\textquotesingle{}s convention and let f be the face on the LEFT of \{v,w\}, g the face on the RIGHT. Note that the ordered pair \{f,g\} is the "{}dual"{} edge to \{v,w\} and geometrically ends up clockwise perpendicular to \{v,w\}, so it points out of f.

For each interior edge e sharing faces f and g, define the "{}directed Mobius edge derivative"{} by d\+M(e)=M(g)\texorpdfstring{$^\wedge$}{\string^}\{-\/1\}.M(f). This is invariant:\+ d\+M(e) is unchanged if we replace P\textquotesingle{} by P"{} = m(\+P\textquotesingle{}), m a Mobius map.   \+The Mobius maps d\+M(e) fix the tangency point within e  and are always parabolic, and we normalize so  trace(d\+M(e))=2 and det(d\+M(e))=1. If d\+M(e) \texorpdfstring{$\sim$}{\string~} \+[a b ; c d]\+,  then the entry c is the "{}(complex discrete) \doxylink{classschwarz_work_1_1_schwarzian}{Schwarzian}"{}  as defined by Orick. More precisely, if z is the tangency  point in the edge e then d\+M(e) has the form      d\+M(e)= \+[1 + c\texorpdfstring{$\ast$}{*}z,-\/c\texorpdfstring{$\ast$}{*}z\texorpdfstring{$^\wedge$}{\string^}2; c, 1-\/c\texorpdfstring{$\ast$}{*}z]\+ \+Moreover, c = s\texorpdfstring{$\ast$}{*}i\texorpdfstring{$\ast$}{*}conj(delta), where delta = exp\{i theta\}  gives the direction of the directed edge e and s is some  real scalar. If eta is the normal to e directed outward  from f (so eta=-\/i\texorpdfstring{$\ast$}{*}delta), then c=s\texorpdfstring{$\ast$}{*}conj(eta). Note that  the complex schwarzian for -\/e is -\/c. (\+This real number s  is distinct from the intrinsic schwarzian discussed below.)  \+INTRINSIC SCHWARZIANS:\+ \+In contrast to complex schwarzians related to mappings  between TWO packings, I am introducing intrinsic schwarzians  associated with edges of a SINGLE tangency packing P.  \+The "{}schwarzian"{} element stored with an edge of P is this  intrinsic schwarzian, a real number defined as above for  maps from a "{}base equilateral"{}. This base equilateral is centered at the origin and has its edge midpoints (tangency points of its circles) at the cube roots of unity. One of our main goals is to understand conditions on those assignments of schwarzians which correspond to circle packings. Also, to understand the relationship between the intrinsic schwarzians of P and P\textquotesingle{} when considering a mapping. See \textquotesingle{}dcel.\+schwarzian.\+java\textquotesingle{}.

This extender is mainly about the mapping case and we store data for both domain and range packing, each having a \textquotesingle{}\doxylink{classutil_1_1_tri_aspect}{Tri\+Aspect}\textquotesingle{} for each face:\+ domain data stored on initiation (or adjusted later), range data stored on demand. Given \{f,g\} in domain and \{F,G\} in range, we have stored the complex schwarzian for the map in \textquotesingle{}domain\+Tri.\+mob\+Deriv\+[.]\+\textquotesingle{}. The intrinsic schwarzians are stored locally in each \doxylink{classutil_1_1_tri_aspect}{Tri\+Aspect} \textquotesingle{}schwarzian\textquotesingle{} and are only moved to \textquotesingle{}Dcel\+Face.\+schwarzian\textquotesingle{} in the packing when requested. Note that the range packing is a global \doxylink{classmath_1_1_mobius}{Mobius} image of domain packing iff all \textquotesingle{}mob\+Deriv\textquotesingle{} are zero iff all intrinsic schwarzians are zero.

A second use of this extender is to experiment with flower layouts using intrinsic schwarzians:\+ see \textquotesingle{}flower\textquotesingle{} command and commands when \textquotesingle{}flower\+Degree\textquotesingle{} is positive. Note that indexing is ticklish.
