\doxysubsection{Detailed Description}
This class supports creation, storage, and manipulations for. 

This class supports creation, storage, and manipulations for combinatorial data on \textquotesingle{}tilings\textquotesingle{}. This data may be associated with a circle packing parent; often, the packing is created from initial tiling data.

An array of \textquotesingle{}\doxylink{classtiling_1_1_tile}{Tile}\textquotesingle{} type is created to maintain the tile information. Each tile will know its vertices (though indices depend on \textquotesingle{}mode\textquotesingle{} and may be changed in various operations). The \textquotesingle{}tile\+Index\textquotesingle{} is maintained by this class and is independent of the vertices. Indexing starts at 1. Each tile has a \textquotesingle{}tile\+Flower\textquotesingle{} with the ccw list of neighbor indices, -\/1 being \textquotesingle{}no neighbor\textquotesingle{}. Vertices are kept in \textquotesingle{}\doxylink{classcombinatorics_1_1komplex_1_1_face_a45c1c8f89779cd8912af2f411aeaf853}{Face.\+vert}\textquotesingle{}.

The tiling and circle packing data are rather independent; we manipulate the tiling combinatorics, for example, with various subdivision processes and only then generate the associated canonical circle packing.

In creating circle packings, tiling data is created depending on \textquotesingle{}mode\textquotesingle{}. For the default \textquotesingle{}mode=3\textquotesingle{}, tiling data is augmented so n-\/gons are formed by 2n barycentrically subdivided triangles, which is the basic combinatorial structure laid out in our "{}conformal tiling"{} definitions. Mode 1 and 2 have less structure.

Since subdivision is one of our main interests, we can build Tiledata to some subdivision depth using subdivision rules. The construction then gives a hierarchy of tilings, stored in \textquotesingle{}graded\+Tile\+Data\textquotesingle{}. This depends on \textquotesingle{}build\+Mode\textquotesingle{}, which records the \textquotesingle{}mode\textquotesingle{} setting of the stored structure, and must be redone if mode is changed. \textquotesingle{}graded\+Tile\+Data\textquotesingle{} is kept ONLY in \textquotesingle{}canonical\+Pack.\+tile\+Data\textquotesingle{}; other instances of \textquotesingle{}\doxylink{classtiling_1_1_tile_data}{Tile\+Data}\textquotesingle{} will have \textquotesingle{}graded\+Tile\+Data\textquotesingle{} set to null. When some depth of a tiling tree is needed, it is put in \textquotesingle{}pack\+Data.\+tiling\+Data\textquotesingle{}, but it does not contain full information. In particular, the full hierarchy cannot be saved to a file, so it must be rebuilt if canonical\+Pack is lost.

\textquotesingle{}graded\+Tile\+Data\textquotesingle{} is generated to some depth in the \textquotesingle{}build\+\_\+sub\textquotesingle{} and \textquotesingle{}subtile\textquotesingle{} commands of \textquotesingle{}\doxylink{classftn_theory_1_1_conformal_tiling}{Conformal\+Tiling}\textquotesingle{}; so \textquotesingle{}canonical\+Pack.\+tile\+Data.\+my\+Tiles\textquotesingle{} has pointers to the leaf tiles in the hierarchy. In general, however, \textquotesingle{}pack\+Data.\+Tile\+Data\textquotesingle{} is pulled out of the hierarchy to some specified depth. If not full depth, then its \textquotesingle{}my\+Tiles\textquotesingle{} are copies (see \textquotesingle{}copy\+Bare\+Bones\textquotesingle{}) only at the specified tile depth. All this depends on \textquotesingle{}mode\textquotesingle{} and must be rebuilt if mode changes.

\doxymbox{\hyperlink{classpacking_1_1_pack_data}{Pack\+Data}} tiling.\+Tile\+Data.\+tiles2\+Full\+Bary (\begin{DoxyParamCaption}\item[{\doxymbox{\hyperlink{classtiling_1_1_tile_data_a7d03700406cae5823579a726b5ba8e3c}{Tile\+Data}}}]{td}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [static]}}



Create a fully realized (built\+Mode 3) packing for given \textquotesingle{}\doxylink{classtiling_1_1_tile_data}{Tile\+Data}\textquotesingle{}. This is a hex refinement of the barycentric refinement of the original tiles:\+ that is,. 

(1) start with tiles as ordered lists of vertices; (2) get triangulation by adding a barycenter to each tile and a barycenter to each tile edge. (3) bary\+\_\+refine:\+ hexrefine the resulting triangles.

This seems complicated, but contends with two problems:\+

(I) Otherwise some vertices would have only 2 neighbors. Now we can handle all tilings from "{}drawings"{}, e.\+g. dessins (II) We now have enough data to follow the tile edges; without the extra vertices, combinatorial geodesics are ambiguous due to the bary\+\_\+refine of the faces in step (3).

A clone of \textquotesingle{}tile\+Data\textquotesingle{} with augmented vertices is attached to the final packing.

Three construction stages:\+ (1) build a packing having a barycenter vert for each tile and for each tile edge; (2) barycentrically subdivide every face; (3) create \textquotesingle{}dual\textquotesingle{}, \textquotesingle{}quad\textquotesingle{}, and \textquotesingle{}wg\+Tiles\textquotesingle{} tile\+Data.

\doxylink{classcombinatorics_1_1komplex_1_1_vertex}{Vertex} \textquotesingle{}mark\textquotesingle{}s in the new packing are set to \{1,2,3\} for \{tile bary\+Vert,original tile corner, new edge vert\}, resp. Each face is becomes a hex flower, and its center is given mark = -\/1.

wg\+Tiles are created for each original tile, also for \textquotesingle{}dual\+Tile\+Data\textquotesingle{} and \textquotesingle{}quad\+Tile\+Data\textquotesingle{}. \textquotesingle{}wg\+Tiles\textquotesingle{} are indexed from 0, type is 1=original, 2=dual, 3=quad. Each has vert\+[3]\+ positively oriented, starting with bary\+Vert (vertex mark=1); the \textquotesingle{}tile.\+mark\textquotesingle{} +-\/ sign shows whether positively or negatively oriented.
