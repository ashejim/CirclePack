\doxysubsection{Detailed Description}
This can be standalone or called from Circlepack. 

This can be standalone or called from Circlepack. It generates \textquotesingle{}html\textquotesingle{} files that accumulate \doxylink{classall_mains_1_1_circle_pack}{Circle\+Pack} \texorpdfstring{$\ast$}{*}.cps (or deprecated \texorpdfstring{$\ast$}{*}.xmd and \texorpdfstring{$\ast$}{*}.cmd) script files in "{}\{parentdirectoryname\}.\+html"{}. This \texorpdfstring{$\ast$}{*}.html file is stored in the parent directory if it is local or in the java.\+io.\+tmpdir if on a web site. The code operates in one of two modes:\+

(1) Can be run standalone with a given URL and file name (optional). (For standalone, run via \textquotesingle{}Script\+Lister.\+jar\textquotesingle{} and specify a url as the argument.) When the \texorpdfstring{$\ast$}{*}html file is loaded into the \doxylink{classall_mains_1_1_circle_pack}{Circle\+Pack} \textquotesingle{}WWW\textquotesingle{} window, it provides info on each script and links for the full path to load it into the \doxylink{classall_mains_1_1_circle_pack}{Circle\+Pack} script window.

In this form of html file there is a ..

entry for each script. It formats 4 pieces of data from the XML-\/encoded script file:\+
\begin{DoxyItemize}
\item Script Name, entered as html comment preceding the table
\item Script header text description (if it exists)
\item Script ABOUT\+\_\+\+IMAGE:\+ (if it exists) and link url (if it exists; default to current directory). Note that the link may have to be adjusted depending on where the script is stored.
\end{DoxyItemize}

(2) When a directory is showing in the WWW window, the "{}\+H"{} button will generate a floating style listing of the scripts with just the About\+Image and name and active link to load. In this case, if the output cannot be stored in an \texorpdfstring{$\ast$}{*}.html file (i.\+e., if it\textquotesingle{}s a remote directory), then it will reside in a file in the java.\+io.\+tmpdir.
