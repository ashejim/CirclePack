\hypertarget{classdcel_1_1_raw_manip}{}\label{classdcel_1_1_raw_manip}\index{dcel.RawManip@{dcel.RawManip}}


This file is for static methods manipulating combinatoric structures. These "{}\texorpdfstring{$\ast$}{*}\+\_\+raw"{} methods typically handle radii and centers only in copying them to new elements and there is almost no dependence on the \doxylink{classpacking_1_1_pack_data}{Pack\+Data} parent. The routines expect \textquotesingle{}vertices\textquotesingle{} to be in place and for bdry edges to be identifiable based on face indices. They may update \textquotesingle{}vertices\textquotesingle{}, but arrays \textquotesingle{}edges\textquotesingle{} and \textquotesingle{}faces\textquotesingle{} are not updated. Red chain may be adjusted, but will be null if there are problems modifying it (sometimes because it is just too complicated). \textquotesingle{}\doxylink{classdcel_1_1_pack_d_c_e_l_a401c93c060269d818d8ef302a52fc22b}{Pack\+DCEL.\+tri\+Data}\textquotesingle{} is set to null, since this data is generally outdated.

\doxysubsection{Detailed Description}
This file is for static methods manipulating combinatoric structures. These "{}\texorpdfstring{$\ast$}{*}\+\_\+raw"{} methods typically handle radii and centers only in copying them to new elements and there is almost no dependence on the \doxylink{classpacking_1_1_pack_data}{Pack\+Data} parent. The routines expect \textquotesingle{}vertices\textquotesingle{} to be in place and for bdry edges to be identifiable based on face indices. They may update \textquotesingle{}vertices\textquotesingle{}, but arrays \textquotesingle{}edges\textquotesingle{} and \textquotesingle{}faces\textquotesingle{} are not updated. Red chain may be adjusted, but will be null if there are problems modifying it (sometimes because it is just too complicated). \textquotesingle{}\doxylink{classdcel_1_1_pack_d_c_e_l_a401c93c060269d818d8ef302a52fc22b}{Pack\+DCEL.\+tri\+Data}\textquotesingle{} is set to null, since this data is generally outdated. 

Typical process:\+ (1) calling routine may \textquotesingle{}zero\+EUtil\textquotesingle{} to zero \textquotesingle{}eutil\textquotesingle{}. (2) calling routine runs \textquotesingle{}{\itshape \+\_\+raw\textquotesingle{} method to modify the dcel structure. (3) the \textquotesingle{}}\+\_\+raw\textquotesingle{} routine should ensure \textquotesingle{}vertices\textquotesingle{} and \textquotesingle{}vert\+Count\textquotesingle{} are updated and space is allocated and ideal faces are created and give negative index. If red chain could not be safely modified in \textquotesingle{}\texorpdfstring{$\ast$}{*}\+\_\+raw\textquotesingle{}, then set \textquotesingle{}red\+Chain\textquotesingle{} null. (4) for any new vertices, preliminary data (cent/\+rad/\+aim, etc.) are typically set from appropriat reference vert. (5) calling routine runs \textquotesingle{}fix\+DCEL\textquotesingle{}, which computes red chain (if needed) and calls \textquotesingle{}Fill\+Inside\textquotesingle{} and \textquotesingle{}attach\+DCEL\textquotesingle{}.

Sometimes \textquotesingle{}\texorpdfstring{$\ast$}{*}\+\_\+raw\textquotesingle{} may store info in \textquotesingle{}\doxylink{classdcel_1_1_pack_d_c_e_l_ab38ff3aa7bafebf781ea90203bab3a16}{Pack\+DCEL.\+old\+New}\textquotesingle{}. Many raw routines can be repeated several times before \textquotesingle{}fix\+DCEL\textquotesingle{} is needed. I\textquotesingle{}m going to try this approach:\+

(a) Before the first \textquotesingle{}\texorpdfstring{$\ast$}{*}\+\_\+raw\textquotesingle{} call, set \textquotesingle{}pack\+DCEL.\+old\+New\textquotesingle{} null.

(b) during each call, compose \textquotesingle{}pack\+DCEL.\+old\+New\textquotesingle{} with local \textquotesingle{}oldnew\textquotesingle{} built during the routine; see \textquotesingle{}\doxylink{classlist_manip_1_1_vertex_map_a723a190279b6cf19f8ac89ea3675c7dc}{Vertex\+Map.\+followed\+By}\textquotesingle{}.

(c) When done, \textquotesingle{}pack\+DCEL.\+old\+New\textquotesingle{} should have \texorpdfstring{$<$}{<}orig,final\texorpdfstring{$>$}{>} pairings which are used during \textquotesingle{}attach\+DCEL\textquotesingle{} to copy data where needed.

Routine to \textquotesingle{}migrate\textquotesingle{} a branch point from interior point v to nghb w. Based on a common geometric method for creating branch points, namely, by attaching two packings along a common slit, the tips of the slits becoming the common branch circle. If the local combinatorics of the two complexes were identical, then the branch point could be modified by extending the slit one more edge on each piece to end at the common neighbors of the original tips; after pasting, the branching would occur at that neighbor. This routine adjusts the combinatorics of the original pasted complex locally to accomplish the same result -- so a branch point v is "{}migrated"{} to a neighbor w. 

Branching is a geometric phenomenon, and while the work is combinatoric, there must exist certain local symmetries to make this work:\+ if \textquotesingle{}edge\textquotesingle{} is \texorpdfstring{$<$}{<}v,w\texorpdfstring{$>$}{>}, the original \textquotesingle{}branch\textquotesingle{} vertex is \textquotesingle{}v\textquotesingle{} and must have even degree (at least 6) and neighbor w must have the same degree as the vertex ww directly opposite of v.

After migration, \textquotesingle{}v\textquotesingle{} remains as the branch vertex, \textquotesingle{}w\textquotesingle{} and \textquotesingle{}ww\textquotesingle{} split the old branch vert edges; no need to adjust aims.
