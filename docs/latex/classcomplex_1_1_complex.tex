\doxysubsection{Detailed Description}
Mutable complex number with arithmetic operations and format conversion. 

This class represents an element of the complex field C, stored in Cartesian form as a pair of doubles (x, y) corresponding to x + iy. It provides the arithmetic expected of a complex-\/number type—addition, multiplication, division, conjugation, square root, exponential, and so on—together with conversions to and from other geometric representations used in Circle\+Pack (Point2D, \doxylink{classmath_1_1_point3_d}{Point3D}, spherical coordinates).

\begin{DoxyNote}{Note}
Instances are {\itshape mutable}:\+ the fields {\ttfamily \doxylink{classcomplex_1_1_complex_a958464318a8e5130f81982811fa26d0a}{x}} and {\ttfamily \doxylink{classcomplex_1_1_complex_a2ea99dadd5a7c4204a6b999f8e826b67}{y}} are public and can be written directly. Most arithmetic methods (plus, times, divide, …) return a {\itshape new} \doxylink{classcomplex_1_1_complex}{Complex} and leave {\ttfamily this} unchanged, but callers should be aware that the underlying data can still be modified through the fields.

Several methods have synonym pairs (e.\+g.~{\ttfamily \doxylink{classcomplex_1_1_complex_a95d34bf26325d45f83602832c07e831a}{plus}} /\+ {\ttfamily \doxylink{classcomplex_1_1_complex_a37cceb4559d9d41156e914717c7fdb98}{add}}, {\ttfamily \doxylink{classcomplex_1_1_complex_ad3d80696b9c43028347a7f7608c78f99}{times}} /\+ {\ttfamily \doxylink{classcomplex_1_1_complex_a6c4b6c2c9e38a886ec44b0a4cf8bf681}{mult}}). Both forms are equivalent; the duplication reflects the code\textquotesingle{}s evolution from C-\/style naming to more Java-\/conventional naming.
\end{DoxyNote}

Divide this complex number by another:\+ z /\+ t.

Parse a string into a \doxylink{classcomplex_1_1_complex}{Complex} number. 

This factory method handles the many formats a user might type when entering a complex number interactively in Circle\+Pack. Accepted formats include (among others):\+
\begin{DoxyItemize}
\item A plain real number or integer, e.\+g. "{}3.\+14"{}, "{}-\/7"{}.
\item A pure imaginary, e.\+g. "{}2i"{}, "{}i"{}, "{}i\texorpdfstring{$\ast$}{*}3"{}, "{}-\/i"{}.
\item A full complex number, e.\+g. "{}1+2i"{}, "{}1 + 2i"{}, "{}3 -\/ 4\texorpdfstring{$\ast$}{*}i"{}, "{}i\texorpdfstring{$\ast$}{*}2 + 3"{}, etc.
\item Circle\+Pack variable references (identifiers starting with \textquotesingle{}\+\_\+\textquotesingle{}), which are resolved at parse time via {\ttfamily \doxylink{classall_mains_1_1_c_p_base_ae881c9a7252175d55b9badfcee0dfbdb}{CPBase\+\#var\+Control}}.
\end{DoxyItemize}

\begin{DoxyParagraph}{Parsing difficulties}
The parser must contend with several ambiguities:\+
\begin{DoxyItemize}
\item \textquotesingle{}i\textquotesingle{} or \textquotesingle{}I\textquotesingle{} can appear before or after the coefficient.
\item A leading \textquotesingle{}-\/\textquotesingle{} might be a negative sign or a Circle\+Pack flag.
\item \textquotesingle{}-\/\textquotesingle{} also appears in scientific notation (e.\+g. "{}3\+E-\/01"{}); the parser takes care not to split on the exponent sign.
\item Optional \textquotesingle{}\texorpdfstring{$\ast$}{*}\textquotesingle{} and \textquotesingle{}+\textquotesingle{} tokens between parts.
\end{DoxyItemize}
\end{DoxyParagraph}
\begin{DoxyNote}{Note}
The method does not yet accept parenthesized formats such as "{}(x, y)"{}.
\end{DoxyNote}
