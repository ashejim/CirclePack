\doxysubsection{Detailed Description}
Hyperbolic circle-\/packing repacker using DCEL data structures. 

Implements Thurston\textquotesingle{}s iterative radius-\/adjustment algorithm for the Poincaré disc model of the hyperbolic plane. Radii are stored internally as "{}x-\/radii"{} (x = 1 − e\texorpdfstring{$^\wedge$}{\string^}\{−2h\} for hyperbolic radius h), which are more numerically stable than hyperbolic radii near 0. During repacking, x-\/radii are temporarily converted to Euclidean labels for the UNM computation, then converted back.

The iteration logic mirrors {\ttfamily \doxylink{classre_pack_1_1_eucl_packer_a8bfd65ca2291ddc0f43bb5e5bfc0ae5e}{Eucl\+Packer\+\#continue\+Riffle}} with the same UNM prediction and superstep acceleration, but with hyperbolic angle-\/sum formulas.

Static up/\+down (or down/\+up) Perron version of old\+Reliable. Not intended for speed, rather to try to ensure monotone behavior of radii. We use the uniform neighbor model (which theoretically avoids overshooting); this also allows us to easily convert computation to euclidean. 

\textquotesingle{}direction\textquotesingle{}:\+ radii go down only, down/\+up in turn, up only, or up/\+down in turn.

E.\+g., up/\+down means increase radii (of adjustable circles) until anglesum \texorpdfstring{$<$}{<}= aim. Then subsequent downward adjustments will, in theory, producing a packing label (at adjustable circles). This \textquotesingle{}up/\+down\textquotesingle{} would be considered our standard "{}downward"{} Perron:\+ the up stage leads to a superpacking, then the downward Perron. CAUTION:\+ may not have enough passes to reach superpacking, then packing.

Return double\+[3]\+:\+ s=anglesum, a=aim \+[0]\+:\+ count of iterations, -\/1 on FAILURE \+[1]\+:\+ l\texorpdfstring{$^\wedge$}{\string^}2 deficiency on upward, sqrt(sum (s-\/a)\texorpdfstring{$^\wedge$}{\string^}2) when (s-\/a)\texorpdfstring{$>$}{>}0 \+[2]\+:\+ l\texorpdfstring{$^\wedge$}{\string^}2 deficiency on downward, sqrt(sum (s-\/a)\texorpdfstring{$^\wedge$}{\string^}2) when (s-\/a)\texorpdfstring{$<$}{<}0 \+[3]\+:\+ total l\texorpdfstring{$^\wedge$}{\string^}2 error:\+ sqrt(sum(s-\/a)\texorpdfstring{$^\wedge$}{\string^}2)

Note:\+ This code could serve as base for an arbitrary precision version in future.

Compute radii of packing to meet angle\+\_\+sum targets, specified as "{}aims"{}. A negative "{}aim"{} means that that circle should not have its radius adjusted. aim of zero is possible only for boundary circles in hyp setting.

All routines are iterative, inspired by routine suggested by Thurston. In hyperbolic case, we use \textquotesingle{}x-\/radii\textquotesingle{}, see description elsewhere. If h is infinite, store eucl radius with negative value for use in graphing its horocycle.

Overlap angles specified \texorpdfstring{$>$}{>} Pi/\+2 can lead to incompatibilities. To avoid domain errors, routines computing angles in such cases return angle Math.\+PI
