Distribute M random ordered points along (a polygonal approximation to) the given Path2\+D.\+Double, return a \doxylink{classcomplex_1_1_complex}{Complex} vector of the points, or null on error. 

Points start with path(0). Choose M consecutive random values from uniform distribution on \+[0,1]\+, accumulate the lengths and scale to length of gpath. (The Mth point is not used, but ensures random spacing with regard to the path\textquotesingle{}s endpoint.)

Note:\+ \textquotesingle{}arclength\textquotesingle{} along gpath is used for positioning the \doxylink{classcomplex_1_1_complex}{Complex} points; it starts with index 0 and last point is index M-\/1.

\doxymbox{\hyperlink{classkomplex_1_1_triangulation}{Triangulation}} random.\+Random\+Triangulation.\+random\+\_\+\+Triangulation (\begin{DoxyParamCaption}\item[{int}]{N}{, }\item[{boolean}]{debug\+\_\+flag}{, }\item[{int}]{hes}{, }\item[{double}]{Aspect}{, }\item[{Path2\+D.\+Double}]{Gamma}{, }\item[{\doxymbox{\hyperlink{classcomplex_1_1_complex}{Complex}}}]{Tau}{}\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [static]}}



This code generates random triangulations of various regions/\+types. Currently allow rectangles, 1-\/tori, regions bounded by curves, or determined by geometry. 

Want to convert these to packings, keeping not only the combinatorics, but also the centers, so that we can generate images, do mappings, etc, based on the triangulation itself. Therefore, it is up to the user to apply further manipulations to the resulting packing.

Arguments (parsed from command in calling routine):\+

(1) N number of interior vertices:\+ in non-\/tori case, 4\texorpdfstring{$\ast$}{*}sqrt(N) additional vertices are added to the boundary of the region or rectangle. If N\texorpdfstring{$<$}{<}12 then set N=12.

(2) debug:\+ if true, then don\textquotesingle{}t call random seed (though it may have been called elsewhere in \doxylink{classall_mains_1_1_circle_pack}{Circle\+Pack}).

(3) hes:\+ geometry:\+ if \texorpdfstring{$>$}{>}0, we triangulate the sphere; choose points randomly in \+[0 1]\+x\+[0 1]\+ and use theta = 2\texorpdfstring{$\ast$}{*}\+Pi\texorpdfstring{$\ast$}{*}x and phi=acos(2\texorpdfstring{$\ast$}{*}y-\/1)

(4) Aspect:\+ Random triangulation of rectangle of this aspect; this takes precedence over hex\texorpdfstring{$>$}{>}0. If Aspect negative, use only interior points, do NOT create additional bdry points.

(5) Gamma:\+ Pathlist defining region.(if non-\/\+NULL, takes precedence over Aspect. Can be on sphere, but don\textquotesingle{}t yet know how (but Gamma always given in 2D cartesian coordinates).

(6) Tau:\+ random triangulation of torus with complex modulus Tau. If z is non-\/zero, this takes precedence over hes, Gamma, and Aspect.
