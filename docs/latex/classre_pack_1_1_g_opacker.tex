\doxysubsection{Detailed Description}
Linearized circle-\/packing algorithm (Collins–\+Orick–\+Stephenson GOpack method). we don\textquotesingle{}t have. 

NOTE:\+ We no longer call this code because we don\textquotesingle{}t have access to sparse matrix computations.

This is intended in the future to provide an implementation through java of Gerald Orick\textquotesingle{}s method for computing circle packings via an iterative routine using Tutte embeddings (currently implemented only in GOPack package for Matlab).

Originally, the plan was to implement these methods in C and then make \doxylink{namespace_j_n_i}{JNI} calls. However, as of 3/\+2022, I\textquotesingle{}ve pulled all the \doxylink{namespace_j_n_i}{JNI} stuff. If we can implement in C, the plan now would be to transfer data in files and use Process\+Builder to execute. I would hope to implement the GOPack methods in Java, but I can\textquotesingle{}t find good java sparce matric routines. (Caution:\+ it appears that things like "{}la4j"{}, for example, are more about compact storage and doesn\textquotesingle{}t implement sparseness in solving systems.)

When this class is instantiated, \textquotesingle{}load\textquotesingle{} creates various persistent data objects. A key design feature is creation on startup of \doxylink{classpacking_1_1_pack_lite}{Pack\+Lite} \textquotesingle{}my\+PLite\textquotesingle{}. This allows packings of portions of larger complexes and it lines up the indexing for sparse matrices. Indexing is now that of \textquotesingle{}my\+PLite\textquotesingle{}.

During computation, data is kept in \textquotesingle{}localradii\textquotesingle{} and \textquotesingle{}localcenters\textquotesingle{}. Latest reliable values are put in my\+PLite \textquotesingle{}radii\textquotesingle{} and \textquotesingle{}centers\textquotesingle{}. When done, transfer data from \textquotesingle{}my\+PLite\textquotesingle{} to \textquotesingle{}p\textquotesingle{}, parent \doxylink{classpacking_1_1_pack_data}{Pack\+Data}.

System has block form \texorpdfstring{$\vert$}{|} A B \texorpdfstring{$\vert$}{|} \texorpdfstring{$\ast$}{*} \texorpdfstring{$\vert$}{|} Z0 \texorpdfstring{$\vert$}{|} = \texorpdfstring{$\vert$}{|} 0 \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} 0 I \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} Zb \texorpdfstring{$\vert$}{|} \texorpdfstring{$\vert$}{|} Zb \texorpdfstring{$\vert$}{|}

A is transition matrix, interior to interior, B is transition matrix, interior to bdry, Z0 is vector of interior radii Zb is vector of bdry radii (do this system once for real, once for imaginary)

We create the right-\/hand-\/side -\/B\texorpdfstring{$\ast$}{*}\+Zb, then solve A\texorpdfstring{$\ast$}{*}\+Z0=-\/B\texorpdfstring{$\ast$}{*}\+Zb for Z0

Calls:\+
\begin{DoxyItemize}
\item \textquotesingle{}start\+Riffle\textquotesingle{} initiates \textquotesingle{}radii\textquotesingle{} to 1 and fills edge weights, but doesn\textquotesingle{}t riffle (i.\+e., does not do any adjustments).
\item \textquotesingle{}restart\+Riffle\textquotesingle{} sets radii to current values from my\+PLite.
\item \textquotesingle{}continue\+Riffle\textquotesingle{} continues with \textquotesingle{}localradii\textquotesingle{} (even if the packing data has been updated).
\item \textquotesingle{}quality\textquotesingle{} returns L2 error in abstract (i.\+e., computed from radii, versus embedded (which is from Tutte-\/type layout).
\item \textquotesingle{}reap\+Radii\textquotesingle{} puts \textquotesingle{}localradii\textquotesingle{} and \textquotesingle{}localcenters\textquotesingle{} into the my\+PLite
\end{DoxyItemize}
