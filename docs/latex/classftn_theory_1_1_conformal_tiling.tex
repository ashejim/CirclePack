\doxysubsection{Detailed Description}
Tilings are combinatorial, locally planar patterns. 

Tilings are combinatorial, locally planar patterns of tiles, the tiles being n-\/gons for n\texorpdfstring{$>$}{>}=1. This extender is for generating, manipulating and displaying tilings via associated circle packings. Tilings may be stored in files, the packing being generated on being read, may be generated by, e.\+g., \textquotesingle{}pave\textquotesingle{}, or may be built in hierarchical fashion from subdivision rules. Much of this code is devoted to this latter scenario.

The basic combinatorics of a tiling are stored in a \textquotesingle{}\doxylink{classtiling_1_1_tile_data}{Tile\+Data}\textquotesingle{} structure, which is a pattern of tiles, each defined by a closed, counterclockwise chain of vertices, starting with the "{}principal"{} vertex, which is \textquotesingle{}vert\+[0]\+\textquotesingle{}.

A major concern involves a family of tilings generated by some finite subdivision rule; see class \textquotesingle{}\doxylink{classtiling_1_1_subdivision_rules}{Subdivision\+Rules}\textquotesingle{}. Calling for some depth of subdivision generates a hierarchy of tilings. This is reflected in the element \textquotesingle{}graded\+Tile\+Data\textquotesingle{} in \textquotesingle{}canonical\+Pack.\+tile\+Data\textquotesingle{}. If \textquotesingle{}pack\+Data.\+tile\+Data\textquotesingle{} has the deepest level tiling, then it is just a pointer to \textquotesingle{}canonical.\+tile\+Data\textquotesingle{}. However, \textquotesingle{}pack\+Data.\+tile\+Data\textquotesingle{} may represent some lesser depth in the hierarchy, in which case its \textquotesingle{}my\+Tiles\textquotesingle{} are bare bones clones at one level in the hierarchy and do not contain full hierarchy information.

A global tiling at a given stratum is defined recursively:\+ the subtilings of the top tiles are found and then pasted together, but those subtilings are pasted together from their subtilings, etc. So one recurses on pasting together tiles until one reaches down to the desired level. It is assumed, in order to do the pastings, that at each stratum, neighboring tiles have edges with the same number of vertices, so they can be pasted. This compatibility is part of the art in defining subdivision rules.

This extender builds the tilings at the full requested depth and creates the associated circle packing as it goes along. It keeps track of the tilings at all intermediate stages in \textquotesingle{}graded\+Tile\+Data\textquotesingle{} as well and it stores all the vertices defining tiles at every stage in the tile\textquotesingle{}s \textquotesingle{}augmented\textquotesingle{} vert list.

There are four levels of combinatorial complexity for packings one can associate with tilings:\+

(1). The minimal has a vertex for each tile corner and a barycenter vertex for each tile. This is mode=1

(2). Next is the barycentric subdivision, which adds a barycenter vert to each edge:\+ this is a standard topological barycentric subdivision of the tiling as a cell complex. This is mode=2. See \textquotesingle{}baryize\textquotesingle{}.

(3). (default) Third is a further barycentric refinement of the barycentric subdivision from (2), mode=3. Each face is further subdivided as a hex flower. This parallels the structure introduced with "{}dessins"{} and is realized in \textquotesingle{}canonical\+Pack\textquotesingle{} for continual reference. Problems in tracing edges (e.\+g., there are three shortest complex edges between two tile corners) is taken care of with the \textquotesingle{}augmented\textquotesingle{} vertex lists.

(4) Finally, there may be various refinements of any of these. Its tiles are displayed using \textquotesingle{}canonical\+Pack\textquotesingle{} information; the link between these can be corrupted --- e.\+g., by swapping indices.

CAUTION:\+ \textquotesingle{}canonical\+Pack\textquotesingle{} has lots of info:\+ the tiles, dual tiles, quad tiles, white/\+grey tiles, etc. This has to be generated from the input \textquotesingle{}\doxylink{classtiling_1_1_tile_data}{Tile\+Data}\textquotesingle{} of tiles alone. This extra info is NOT saved in packing files --- only the original tiles are saved. Thus, to use this extra info one must always reconstruct it using this pack extender.

When constructing subdivision tilings to some depth d, intermediate tilings stored in \textquotesingle{}tile\+Data.\+graded\+Tile\+Data\textquotesingle{}; see \textquotesingle{}set\+\_\+\+TD\textquotesingle{}. \doxylink{classtiling_1_1_tile}{Tile} \textquotesingle{}vert\textquotesingle{}s and \textquotesingle{}aug\+Vert\textquotesingle{}s are all in terms of circles of the parent packing. In particular, \textquotesingle{}mode\textquotesingle{} should be set initially, it can\textquotesingle{}t really be reset without regenerating the packing from scratch.

We create and save \textquotesingle{}canonical\+Pack\textquotesingle{}; if there is no tiling, then a packing with \doxylink{classtiling_1_1_tile_data}{Tile\+Data} can be loaded or a tiling generated using \textquotesingle{}pave\textquotesingle{}. \textquotesingle{}\texorpdfstring{$\vert$}{|}ct\texorpdfstring{$\vert$}{|} export -\/c \{p\}\textquotesingle{} will export \textquotesingle{}canonical\+Pack\textquotesingle{} to pack \textquotesingle{}p\textquotesingle{}, \textquotesingle{}\texorpdfstring{$\vert$}{|}ct\texorpdfstring{$\vert$}{|} get\+\_\+canon\textquotesingle{} will replace parent \textquotesingle{}pack\+Data\textquotesingle{} with a copy of \textquotesingle{}canonical\+Pack\textquotesingle{}.

The aim of this extender is to create/\+modify tilings, generate new packings which can replace \textquotesingle{}pack\+Data\textquotesingle{} or be copied or saved, and generate associated dual, quad, and grey/\+white tilings.

CAUTION:\+ \textquotesingle{}canonical\+Pack\textquotesingle{} is fixed after it is formed, but \textquotesingle{}pack\+Data\textquotesingle{} or its copy of \textquotesingle{}\doxylink{classtiling_1_1_tile_data}{Tile\+Data}\textquotesingle{} might be changed. The idea is to use \textquotesingle{}canonical\+Pack\textquotesingle{}, its \textquotesingle{}\doxylink{classtiling_1_1_tile_data}{Tile\+Data}\textquotesingle{}, and \textquotesingle{}\doxylink{classtiling_1_1_tile_data_ab6ec68fab86e9d968abdb5ca6cd0c6af}{Tile\+Data.\+graded\+Tile\+Data}\textquotesingle{} as a base to spawn and modify other tilings. \textquotesingle{}put\+\_\+canon\textquotesingle{} will recreate \textquotesingle{}canonical\+Pack\textquotesingle{}.
