# CirclePack Geometry and Layout Extras

## About this page

Beyond `repack`, `layout`, and `max_pack` (the core solvers), CirclePack
has a collection of commands for geometric manipulation and layout
control: Möbius transformations, scaling and rotation, canonical
normalizations, switching geometries, hex-refinement, holonomy, and
view control.

This page groups those commands by theme. Individual commands with
substantial complexity — such as `layout`, `repack`, `max_pack` — have
their own pages; this page covers the remaining geometry-adjacent
machinery.

Commands covered: `Mobius`, `inv_Mobius`, `appMob`, `pair_mob`,
`cir_invert`, `rotate`, `scale`, `scale_aims`, `scale_radii`,
`norm_scale`, `NSpole`, `focus`, `alpha`, `gamma`, `geom_to_h`,
`geom_to_e`, `geom_to_s`, `dual_layout`, `holonomy_trace`,
`hex_refine`, `hex_slide`, `hh_path`, `flat_hex`, `face_err`,
`square_fit` (`sq_fit`).

## Möbius transformations

A **Möbius transformation** is `z ↦ (az + b)/(cz + d)` — the natural
symmetry group of circle packings. Applying one carries circles to
circles, preserves tangencies, and (in the orientation-preserving
case) preserves overlap angles. CirclePack has multiple commands for
applying Möbius transformations to a packing.

### `Mobius` / `mobius` — Apply the stored transformation

```
Mobius                           # apply CPBase.Mob to all vertices
Mobius {v..}                     # apply to specific vertices
Mobius -s {v..}                  # also apply, but skip side-pair updates
```

Reads `CPBase.Mob` (set earlier with `set_Mobius` or generated by
some other command) and applies it to the listed vertices. Default
is "all vertices."

The `-s` flag suppresses modification of side-pairing maps — useful
when you're applying a transformation to visualize it but don't want
to permanently reparameterize a multiply-connected surface.

### `inv_Mobius` / `inv_mobius` — Apply the inverse

```
inv_Mobius                       # apply CPBase.Mob with reversed orientation
inv_Mobius {v..}                 # ditto, specific vertices
```

Same as `Mobius` but applies the orientation-reversed version of
`CPBase.Mob`. This is *not* the mathematical inverse — it's the
same transformation with the `oriented` flag flipped. For the true
inverse, compute it with `set_Mobius` first.

### `appMob` — Apply an inline Möbius

```
appMob a.re a.im b.re b.im c.re c.im d.re d.im [flip]
```

Applies a Möbius transformation defined directly in the command,
without first storing it. Takes 8 real numbers (the real and
imaginary parts of a, b, c, d in the standard matrix form), plus an
optional 9th value: if negative, orientation is reversed.

Always applies to all vertices (`"a"` list). For partial
application, use `set_Mobius` followed by `Mobius {v..}`.

Example:
```
appMob 1 0 0 0 0 0 1 0           # identity (no-op)
appMob 0 0 0 1 1 0 0 0           # z → i/z
```

### `pair_mob` — Apply a named side-pairing Möbius

```
pair_mob {label}
pair_mob -s {label}
```

Looks up a side-pairing transformation by its **label** (a string
assigned during side-pairing construction in a multiply-connected
packing) and applies it to all vertices.

The `-s` flag is the only flag accepted; currently it's effectively
decorative in the current parser. The real argument is the label.

Errors if the label doesn't match any stored side-pair.

Useful for stepping through the universal cover of a torus or
higher-genus surface by composing successive side-pair maps.

## Rotation, scaling, normalization

These commands apply simple Euclidean transformations to the packing.

### `rotate {x}` — Rotate by `x·π` radians

```
rotate 0.5                       # rotate by 90 degrees
rotate 1                         # rotate by 180 degrees
rotate -0.25                     # rotate by -45 degrees
```

Takes a single real number and rotates the entire packing by that
number times π. So `rotate 1` is a half turn; `rotate 0.5` is a
quarter turn. The packing is updated in place.

### `scale {factor}` — Euclidean scale

```
scale 2                          # double the packing
scale 0.5                        # halve
scale                            # default factor 1.1 (grow by 10%)
```

Multiplies every radius and every center's magnitude by `factor`.
Euclidean only (hyperbolic scaling is different and uses the
Möbius command instead).

Accepts a positive factor; non-positive values are rejected.

### `scale_aims {factor} {v..}` — Scale aims

```
scale_aims 2 a                   # double every aim
scale_aims 0.5 i                 # halve interior aims
scale_aims 1.5 Iv 5              # tripling aims around vertex 5
```

Multiplies the aim at each listed vertex by `factor`. Modifies
targets; doesn't re-solve. Follow with `repack; layout` to see the
effect.

### `scale_rad -q{p} {factor} {v..}` — Scale radii against another pack

```
scale_rad -q1 2 a                # double radii from pack 1's radii
```

Sets each vertex's radius to `factor × (pack p's radius at v)`.
Useful for moving between related packings or for generating
controlled perturbations.

### `square_fit` (alias: `sq_fit`) — Fit packing to unit square

```
square_fit
```

Euclidean only. Scales the packing so that it fits within the unit
square `[-1, 1] × [-1, 1]` — specifically, so the maximum of
|x| + r or |y| + r over boundary vertices equals 1. Does not
recenter; just scales.

Useful for producing normalized figures for publication.

## Canonical normalizations (`norm_scale`)

`norm_scale` has a rich set of sub-modes for bringing a packing
into a canonical position. All sub-modes are invoked via a single
flag letter.

### Synopsis

```
norm_scale -a {area}             # scale to given area (Eucl only)
norm_scale -c {v} {r}            # scale so vertex v has radius r
norm_scale -e {q} {v} {w}        # match v's radius to w's radius in pack q
norm_scale -h {v} {w}            # align v → w along positive x-axis
norm_scale -i {v}                # put vertex v at z = i
norm_scale -u {v}                # put vertex v on unit circle
norm_scale -U                    # scale down to fit unit disc
norm_scale -m {u v x1 y1 x2 y2}  # Möbius sending u → (x1,y1), v → (x2,y2)
norm_scale -s                    # schwarzian-flower normalization
norm_scale -t                    # torus: normalize and report modulus τ
```

### Sub-modes in detail

| Flag    | Meaning |
|---------|---------|
| `-a {x}` | Scale so total face area equals `x`. Eucl only. |
| `-c {v} {r}` | Scale so vertex `v` has radius `r`. |
| `-e {q} {v} {w}` | Scale so vertex `v` in this pack matches radius of vertex `w` in pack `q`. |
| `-h {v} {w}` | Rotate so the vector from `v` to `w` lies on the positive x-axis (horizontal). |
| `-i {v}` | Rotate and scale so vertex `v`'s center is at `z = i`. |
| `-u {v}` | Scale so `v`'s center lies on the unit circle. |
| `-U`    | Scale down so boundary fits in unit disc. Only scales down; no action if already inside. Eucl only. |
| `-m {u v x1 y1 x2 y2}` | Apply Möbius transformation sending `u`'s center to `(x1, y1)` and `v`'s center to `(x2, y2)`. |
| `-s`    | Schwarzian-flower normalization — for single-flower packings, centers the flower with schwarzian-based orientation. |
| `-t`    | Torus only. Normalizes the packing and reports the modulus `τ`. |

### Examples

```
norm_scale -a 1.0                # scale to unit area
norm_scale -c 5 0.1              # vertex 5 gets radius 0.1
norm_scale -h 1 2                # vector 1 → 2 is horizontal
norm_scale -i 1                  # vertex 1 lives at i
norm_scale -u 5                  # vertex 5 lives on unit circle
norm_scale -U                    # ensure boundary fits in unit disc
norm_scale -m 1 5 0 0 1 0        # Möbius: vert 1 → 0, vert 5 → 1
norm_scale -t                    # for a torus, report τ
```

### Notes

- `-s` (schwarzian) works only when `packData.intNodeCount == 1`
  — i.e., for a single-flower test packing.
- `-t` requires the packing to be a 1-torus (genus 1, no boundary).
- `-U` never scales up — if your boundary is already inside the
  disc, the command is a no-op.
- `-h`, `-i`, `-u`, `-m` are the usual "set up for a figure"
  sequence.

## Spherical normalization (`NSpole`)

For spherical packings, `NSpole` normalizes so that the **centroid
of tangency points** (or optionally centers) is at the origin. The
name reflects the convention that it positions the pack relative
to north and south poles.

### Synopsis

```
NSpole                           # default: centroid of tangency points
NSpole -x t {cycles}             # experimental: tangency, n cycles
NSpole -x c {cycles}             # experimental: centers, n cycles
```

### Default behavior (no flags)

1. Detects whether the packing has non-default inversive distances
   (overlap packing).
2. For tangency packings: compute centroid of tangency points,
   construct a sphere Möbius normalizer, apply it. (One pass.)
3. For overlap packings: iterate — circle centers are not Möbius
   invariant, so the computation is repeated (up to 5 cycles) until
   the centroid is within tolerance of zero.

### `-x` experimental mode

Lets you explicitly choose tangency (`t`) or centers (`c`) and
specify cycle count. The default cycle count in experimental mode
is 5.

### Requirements

- Must be spherical (`hes > 0`). Errors otherwise.
- Falls back to `CPI_CP_PackingUtility.normalize` (Orick's code)
  on failure.

### Examples

```
NSpole                           # usual normalization
NSpole -x c 10                   # up to 10 cycles using centers
```

## Inversion

### `cir_invert` — Invert in a circle

```
cir_invert {v1} {v2}             # invert in the circle of v1, fixing v2
cir_invert -u {v1}               # invert in the unit circle
```

Applies Möbius inversion:

- With two vertices: invert in circle `v1`, with `v2` as the anchor
  (its center determines how to orient the inversion).
- With `-u {v1}`: invert in the unit circle itself; `v1` serves as
  the anchor for orientation decisions.

The underlying machinery constructs the Möbius transformation
`Mobius.cir_invert(ctr1, rad1, ctr2, rad2)` and applies it to all
vertices.

Converts from hyperbolic to Euclidean before applying (hyperbolic
data is converted to Euclidean representations for the inversion
computation).

### Examples

```
cir_invert 1 5                   # invert in vert 1's circle, using 5 as anchor
cir_invert -u 1                  # invert in unit circle, using 1 as anchor
```

Useful for constructing reflections, for moving a region of the
packing outside the unit disc, or for studying dynamics of
circle-packing maps.

## Root vertices and view control

### `alpha {v}` — Set root vertex

```
alpha 5                          # make vertex 5 the new alpha
```

Sets vertex `v` as the packing's new `alpha` — the root vertex,
placed at the origin during layout. Also updates face drawing order
if needed.

See **concepts** for the role of alpha in CirclePack.

### `gamma {v}` — Set orientation vertex

```
gamma 12                         # make vertex 12 the new gamma
```

Sets vertex `v` as the new `gamma` — placed on the positive y-axis
during layout. Immediately relayouts.

### `focus` — Center the canvas view on a vertex or point

```
focus                            # center on active vertex
focus {v}                        # center on vertex v
focus -v {v}                     # same, explicit flag
focus -vs {x} {v}                # center on v AND zoom so v's diameter fills x
focus -z {x} {y}                 # center canvas at complex point (x,y)
```

Moves the canvas viewbox so the target is centered. Unlike `Mobius`
or `rotate`, this does not modify packing data — it only adjusts
the view. Triggers a redraw (`disp -wr`).

Particularly useful in interactive work:

```
focus -vs 10 5                   # zoom in on vertex 5, occupying 10x its radius
```

## Geometry conversion

CirclePack packings can live in hyperbolic, Euclidean, or spherical
geometry. The `geom_to_*` commands change the current geometry
code (`hes`).

### Synopsis

```
geom_to_h                        # convert to hyperbolic
geom_to_e                        # convert to Euclidean
geom_to_s                        # convert to spherical
geom_to_h -l                     # same, leaving radii unchanged
```

### The `-l` flag

By default, `geom_to_*` may convert radii appropriately for the
target geometry. With `-l`, the stored radii are preserved
bit-for-bit — useful when you want to swap geometries without
disturbing your work.

### What gets touched

- Geometry code (`hes`).
- Radii (by default, converted; preserved with `-l`).
- The curvature table (`fillcurves` is called for `h` and `e`).
- The displayed geometry in the GUI.

Packings do not automatically switch geometries — these commands
are explicit.

### Examples

```
geom_to_e                        # switch to Eucl; radii translated
geom_to_h -l                     # switch to hyp; keep radii literal
geom_to_s                        # switch to sphere
```

### Notes

- Conversion **does not** re-solve the packing. After changing
  geometry, you'll typically want `repack; layout` to get a valid
  embedding.
- `geom_to_s` is a common step before `max_pack` for
  sphere-topology packings.

## Dual layout

### `dual_layout`

```
dual_layout
```

Lays out the packing using the **dual** structure: converts to
spherical, sets the first face of `layoutOrder` as an equilateral
triangle with radius 0.25, then lays out following `layoutOrder`
using single-edge placement (rather than averaging across all
available edges).

This is the modern replacement for the older `sch_layout` command
(which has been retired).

Used in experiments where the dual graph structure matters — for
instance, schwarzian-deformation visualizations that need a
sphere-based reference layout.

## Holonomy

### `holonomy_trace` — Report holonomy around a loop

For multiply-connected packings, compose the side-pairing Möbius
transformations around a closed loop and report the trace (or
write it to a file).

### Synopsis

```
holonomy_trace -s {n}                     # holonomy around side-pair n
holonomy_trace {f..}                      # holonomy around a face sequence
holonomy_trace -s {n} -f {filename}       # also save to file
holonomy_trace {f..} -a {filename}        # append to file
```

### Two argument forms

- **Side-pair form** (`-s {n}`): walks the `n`th side of the
  fundamental domain, composing the side-pair maps to produce the
  resulting Möbius. Returns its trace.
- **Face-list form** (`{f..}`): walks through a user-specified
  sequence of faces, finding the half-edge from each face to the
  next, building a `HalfLink`, and computing holonomy along it.
  Successive faces must be contiguous (share an edge).

### Output

- To the message pane by default.
- If `-f {filename}` or `-a {filename}` is supplied, also writes
  to the file (append with `-a`, script with `-s` per the normal
  file-flag convention).

### Examples

```
holonomy_trace -s 1              # holonomy of side-pair 1
holonomy_trace 1 2 3 4           # holonomy around faces 1→2→3→4
holonomy_trace -s 1 -f data.txt  # save to file
```

### Notes

- A trace far from 2 indicates a non-closing packing — use this
  as a numerical diagnostic for multiply-connected packings.
- First two faces in the face-list form must be contiguous, or
  `holonomy_trace` throws an error.

## Hex manipulation

Several commands operate specifically on hexagonal structure
(degree-6 vertices) — common in maximal and highly symmetric
packings.

### `hex_refine` — Hex-refine

```
hex_refine
```

Refine every face by adding a barycenter to each edge and to each
face interior, producing a finer triangulation where the original
vertices are preserved. The result has four times as many faces.

No arguments. Returns the new node count on success.

### `hex_slide {v w}` — Slide along a hex axis

```
hex_slide 3 5
```

Along the hex chain passing through edge `(v, w)`, apply a
"right-slide" move — a combinatorial modification related to hex
refinement. Currently applies to one edge at a time.

Errors if the edge doesn't lie on a simple closed hex axis.

### `hh_path` — Half-hex path construction

```
hh_path [flags] {e..}
```

Build a path along the half-hex structure through a given edge
list. The stopping condition is controlled by flags:

| Flag       | Meaning |
|------------|---------|
| `-a`       | Append to given edge path (input is the path being extended). |
| `-b`       | Stop at boundary. |
| `-c`       | Continue — no stopping condition. |
| `-x`       | Stop when the path crosses a vertex already hit. |
| `-S`       | Stop when the path meets and lines up with the original edge. |
| `-N {n}`   | Length limit: add at most n edges. |

Default stopping condition (`-b -S -x` combined) is reasonable for
most uses.

The computed path is stored as `packData.elist`. Returns the
path's size.

### `flat_hex {v..}` — Make a flat hex region

```
flat_hex a
flat_hex i
```

For listed vertices in a hex-refined packing, adjust radii so the
local packing is flat (angle sum = 2π) — effectively making those
vertices and their neighborhoods conformally Euclidean.

Returns the number of vertices adjusted.

## Quality diagnostics

### `face_err {x} {f..}` — Count misplaced faces

```
face_err 1.5                     # all faces with error > 1.5
face_err 1.2 a                   # same, explicit list
face_err 2.0 Iv 5                # faces around vertex 5
```

Counts how many faces have a placement error exceeding the
threshold `x` (which must be > 1.0). A face's "error" is the ratio
of its actual angle to its intended angle; values near 1 are good,
far-from-1 values indicate layout problems.

Reports the count to the message pane.

Related: `?qual -o {f..}` — orientation errors. See the
**queries** reference.

## Typical workflows

### Move a packing to a canonical position

```
max_pack
repack
layout

# Put vertex 1 at origin, vertex 5 on positive x-axis, scale to unit area
alpha 1                          # vertex 1 as root
norm_scale -h 1 5                # make 1→5 horizontal
norm_scale -a 1.0                # scale to unit area
disp -wr
```

### Apply a Möbius transformation interactively

```
# Define the Möbius sending three points to three other points
set_Mobius -xyzXYZ  0 0  1 0  0 1   0 0  2 0  0 2    # doubles
Mobius                           # apply to all vertices
disp -wr
```

### Reflect a packing

```
# Invert in the unit circle, anchoring vertex 1
cir_invert -u 1
disp -wr
```

### Zoom in on a feature

```
focus -vs 15 37                  # center on vertex 37, zoom to 15x radius
disp -wr
```

### Check a torus packing closes up

```
max_pack
repack
layout
holonomy_trace -s 1              # trace should be near 2
holonomy_trace -s 2              # and again
```

### Switch geometries for visualization

```
# Start with a hyperbolic packing
max_pack

# Show it on the sphere
geom_to_s -l                     # preserve radii
layout
disp -wr
```

### Refine then reflatten

```
hex_refine                       # refine all faces
set_aim_default                  # new vertices start with aim 2π
repack
layout
flat_hex a                       # ensure everything is truly flat
```

## Notes and gotchas

- **`rotate` takes radians in units of π.** `rotate 0.5` means 90°,
  not 0.5 radians.
- **`scale` defaults to 1.1**, not 1.0. If you type `scale` with no
  argument you'll grow the packing by 10%.
- **`Mobius` with no arguments applies to all vertices** — same
  as `Mobius a`.
- **`inv_Mobius` is not the true inverse** — it's the
  orientation-reversed version of `CPBase.Mob`. Use `set_Mobius`
  with the actual inverse matrix if you need mathematical inverse
  behavior.
- **`norm_scale -t` requires a 1-torus.** It errors on anything else.
- **`norm_scale -s` (schwarzian flower) requires exactly one
  interior vertex.** Used for theoretical single-flower tests.
- **`cir_invert` converts hyperbolic to Euclidean before inverting**
  — the result leaves the packing in Euclidean. If you started
  hyperbolic and want to stay hyperbolic, follow with `geom_to_h -l`.
- **`geom_to_*` does not re-solve.** Always follow with
  `repack; layout` to get a valid embedding in the new geometry.
- **`NSpole` is spherical-only.** Errors on Euclidean or
  hyperbolic packings.
- **`focus` changes the view, not the packing.** Unlike `Mobius`,
  nothing in the stored packing data changes — just the canvas
  viewbox.
- **`alpha` and `gamma` both trigger relayout.** The face drawing
  order and normalized positions depend on them, so a change
  forces a new layout.
- **`pair_mob` requires named side-pairs to exist.** On a
  simply-connected packing, there are none and it errors.
- **`dual_layout` always converts to spherical.** If you want a
  non-sphere layout using schwarzians, use `layout -s` directly.
- **`holonomy_trace` face-list form is picky.** The first two
  faces must be contiguous; otherwise it errors.

## Source

- `packing.PackData.apply_Mobius` — the underlying Möbius
  application routine used by `Mobius`, `inv_Mobius`, `appMob`,
  `pair_mob`, `norm_scale -m`, `cir_invert`.
- `math.Mobius` — Möbius transformation class, including
  `cir_invert`, `mob_abAB`, `mob_xyzXYZ` constructors.
- `packing.PackData.rotate`, `eucl_scale`, `scale_aims`, `scale_rad`,
  `flat_hex`, `hex_refine`, `right_slide` — direct packing-modification
  methods.
- `geometry.NSpole` — the spherical normalization class; see
  `parseNSpole` for flag handling.
- `rePack.HexPaths.halfHexPath` — `hh_path` implementation.
- `dcel.PackDCEL.setAlpha`, `setGamma`, `layoutPacking` — rooting
  and layout.
- `packing.PackData.geom_to_h`, `geom_to_e`, `geom_to_s` — geometry
  conversion.
- `packing.QualMeasures.count_face_error` — `face_err` backend.
- `util.ViewBox.focusView`, `setWidthHeight` — `focus` backend.
- `input.CommandStrParser` — top-level dispatch for each of these
  commands.
