136 /* Rewrote the code below to fix the freezing issue.
137 * When we start a process, we're responsible for its
138 * standard streams (output, error, and input). Depending
139 * on the specific system, these streams have limited
140 * buffer sizes. In the original code, standard error
141 * isn't handled. The buffer would eventually fill, and
142 * the next time CirclePack attempted to write to standard
143 * error, the call would hang until whoever is on the
144 * other end of the stream got around to handling it.
145 * This is us, and we never did so. Killing the launcher
146 * probably handed control of the streams over to the
147 * JVM or OS, which promptly flushed the buffers and kept
148 * them clear, allowing CirclePack to continue.
149 *
150 * In the new code, I spin two threads which forward the
151 * streams appropriately. Threading is necessary to avoid
152 * certain deadlock conditions, like standard output
153 * filling up while we're waiting to read from standard error.
154 *
155 * Java 1.7 has methods to handle this redirection built-in,
156 * so we could do all of this in two lines, but 1.7 is
157 * cutting-edge and the feature is relatively minor so I
158 * just did it "by hand" to avoid compatibility issues for
159 * users with older JVMs.
160 /*
161 System.out.println("Waiting on process");

176 // Get CirclePack's standard output and error as something we can read from.
177 final BufferedReader circlePackOut =
178 new BufferedReader(new InputStreamReader(p.getInputStream()));
179 final BufferedReader circlePackErr =
180 new BufferedReader(new InputStreamReader(p.getErrorStream()));
181
182 // Create threads that forward CirclePack's standard output and
183 // error to this launcher's standard output and error.
184 Thread outputForwarder = new Thread() {
185 public void run() {
186 String line;
187 try {
188 while ((line = circlePackOut.readLine()) != null)
189 System.out.println(line);
190 circlePackOut.close();
191 } catch (IOException e) {
192 System.err.println(
193 "RunCirclePack: Error forwarding standard output. Standard output will no longer be forwarded.");
194 }
195 }
196 };

204 } catch (IOException e) {
