Detailed Description

A point on the Riemann sphere with both "real" and "apparent" coordinates.

Extends Complex to represent a point on the unit sphere that carries two coordinate systems simultaneously:

  • Real spherical coordinates (theta, phi) — the mathematical position on the sphere.
  • Apparent coordinates — the screen-projected position after applying a Matrix3D orthogonal transformation that encodes the current viewing orientation.

The inherited (x, y) fields hold the apparent (theta, phi) after the viewing transformation is applied in the constructor.

Note
The apparent-sphere coordinates are ephemeral; they must be recomputed whenever the viewing orientation changes. The onFront flag indicates whether the point is on the hemisphere facing the viewer.
Author
Ken Stephenson

◆ matrix3Dz()

Complex complex.SphPoint.matrix3Dz ( boolean toA,
Matrix3D trans,
Complex sph_pt )
static

Apply an orthogonal transformation to convert between real and apparent spherical coordinates.

When toA == true, the transformation maps a real (theta, phi) point to its apparent-sphere (theta, phi) using the orthogonal matrix trans. When toA == false, the inverse (transpose) is applied, mapping apparent → real coordinates.

The algorithm proceeds in three steps:

  1. Convert (theta, phi) → Cartesian (x, y, z) on the unit sphere.
  2. Multiply by trans (or its transpose).
  3. Convert the resulting (x, y, z) back to (theta, phi).
Note
This method may no longer be called directly; see toApparentSph and toRealSph in SphView.java.
Parameters
toAtrue to go to the apparent sphere; false for the reverse.
transThe 3 × 3 orthogonal viewing-orientation matrix.
sph_ptInput spherical coordinates as a Complex (theta, phi).
Returns
A new Complex holding the transformed (theta, phi).
