Detailed Description

/** For opening/augmenting/closing postscript output files. For opening/augmenting/closing postscript output files. Status of 'fp' is important for opening/closing/adding decisions, etc. This manager persists, but 'fp' changes.

This is all rather complicated since stuff can be added to the postscript file in increments, pages can be defined, material inserted, etc. The final file is created only when it is about to be closed: then we open the file and

  1. add prefix material (where, eg. 'BoundingBox' may be adjusted for text at end); see 'initialize'
  2. copy in the main body, temporary in file, 'bodyFile'; this is where the 'fp' file pointer is set and where 'postFactory' writes stuff.
  3. copy in trailer stuff accummulated in 'tail' file.
  4. close with any text accummulated in 'textBuffer'.

Also, there are questions of interaction with user and "PostScript" tab, errors to catch, filename, where to store.

See http://www.cs.otago.ac.nz/cosc463/random-access.htm for hints
