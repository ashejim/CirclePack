\doxysubsection{Detailed Description}
Extender for managing "{}conformal welding"{} manipulations. 

Extender for managing "{}conformal welding"{} manipulations.

TODO:\+ currently converting to DCEL while also generalizing to work with non-\/simply connected situations. 1/\+2022.

To "{}weld"{} is to attach two packings to one another along part or all of their boundaries. The attachment rules involve three maps:\+ m1 (resp. m2) is a map from the relevant segment of vertices in bdry p1 (resp. p2) to [0,1] and h is a welding "{}map"{}, a piecewise continuous map h:\+ [0,1] -\/$>$ [0,1].

Typically, m1 (or m2) might be (normalized) harmonic measure (w.\+r.\+t. some interior point) or eucl length. Eventually, it could be explicitly specified. Also might eventually attach multiply-\/connected complexes along more that one boundary component.

FOR NOW:\+
\begin{DoxyItemize}
\item weld one component at a time
\item welds occur along full boundary components
\item eucl/\+sph case:\+ use edge lengths
\item hyperbolic case:\+ bdry components must be fully
\item ideal vertices or fully finite vertices. In the
\item former case, harmonic measure w.\+r.\+t. the origin
\item (i.\+e., arc length on the unit circle)
\end{DoxyItemize}

Pasting requires alignment of the two boundary\textquotesingle{}s vertices, so there is a process to match/\+add vertices. ~\newline


After the welding, weld-\/vertices are in vlist and elist. Depending on options, \textquotesingle{}adjoin\textquotesingle{} is carried out here or one can call it separately now that bdrys have the same numbers of vertices.

Objects involved:\+

We maintain local packings p1 and p2. We always weld p1 onto p2. That is, we assume the welding map \textquotesingle{}h\textquotesingle{} takes the bdry component of p1 onto that of p2. In simply connected case, p1 plays role of interior of unit disc, p2 the exterior. The convention in the welding map is counterclockwise around bdry of p1 and clockwise around p2 (opposite to the convention in \textquotesingle{}adjoin\textquotesingle{}).

welding map:\+ real function h:\+[0,1]-\/-\/$>$[0,1] (reflecting a homeomorphism of the circle).

welding list:\+ Intermediate data which specifies how vertices on the two packings are matched/\+augmented. For now this is stored in \textquotesingle{}/\+tmp/\+weld\+List\+\_\+xxxxx.w\textquotesingle{} Format:\+ Vv Nv Vn Nn ... Vv First character, capitalized, refers to bdry of p, the second, lower case, bdry of q. Letters V/\+v represent original verts, N/\+n represent new added verts.

So reading through the file, a \textquotesingle{}Vv\textquotesingle{} means an existing vertex of p should weld to an existing vertex of q. \textquotesingle{}Nv\textquotesingle{} means a new vertex should be added to p and welded to an existing vertex of q; conversely for \textquotesingle{}Vn\textquotesingle{}. (note:\+ original two first verts should be identified, as should be original two last vertices, so list should start Vv and end Vv.)

Here are routines involved; some could be called for specific tasks, but generally they are parts underlying a single processing call.

readweldmap:\+ read a welding map into a Pathlist variable (generally temporary). Format for data is PATH x1 y1 x2 y2 ... xn yn END

weld\+\_\+map\+\_\+to\+\_\+list:\+ this is the main workhorse routine. It reads a real-\/valued welding map from a file and stores the welding list of V\textquotesingle{}s and N\textquotesingle{}s.

weld:\+ This actually walks around the boundaries and figures out how to match/\+augment the vertices based on a given welding list.

add\+\_\+between:\+ utility to add verts when necessary, fix flowers, etc. This actually changes its target packing, so if things go wrong, that packing may be damaged.

weld\+Using\+Map:\+ function normally called to gather all these operations together. Need to give it the filename for a welding map. It ends up adjusting p and q; results land in p (\textquotesingle{}pack\+Data\textquotesingle{}). \textquotesingle{}vlist\textquotesingle{} gives the welded vertices.
