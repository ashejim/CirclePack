\doxysubsection{Detailed Description}
Doubly-\/connected edge list (DCEL) with circle-\/packing geometry data. 

This is the primary combinatorial data structure for a circle packing. It stores the triangulation as arrays of {\ttfamily \doxyref{combinatorics.\+komplex.\+Vertex}{p.}{classcombinatorics_1_1komplex_1_1_vertex}}, {\ttfamily \doxyref{combinatorics.\+komplex.\+Half\+Edge}{p.}{classcombinatorics_1_1komplex_1_1_half_edge}}, and {\ttfamily \doxyref{combinatorics.\+komplex.\+Dcel\+Face}{p.}{classcombinatorics_1_1komplex_1_1_dcel_face}} objects, together with the "{}red chain"{} ({\ttfamily \doxyref{combinatorics.\+komplex.\+Red\+Edge}{p.}{classcombinatorics_1_1komplex_1_1_red_edge}}) that encodes boundary and side-\/pairing information for multiply-\/connected surfaces.

\begin{DoxyNote}{Note}
The DCEL representation replaced an older adjacency-\/list model. Many legacy routines in {\ttfamily \doxyref{packing.\+Pack\+Data}{p.}{classpacking_1_1_pack_data}} delegate to this class for combinatorial queries.
\end{DoxyNote}
\begin{DoxySeeAlso}{See also}
\doxyref{Comb\+DCEL}{p.}{classdcel_1_1_comb_d_c_e_l} Static combinatorial construction and manipulation routines. 

\doxyref{Raw\+Manip}{p.}{classdcel_1_1_raw_manip} Low-\/level DCEL surgery (flips, insertions, deletions). 

\doxyref{packing.\+Pack\+Data}{p.}{classpacking_1_1_pack_data} The top-\/level \doxyref{packing}{p.}{namespacepacking} container that owns a \doxyref{Pack\+DCEL}{p.}{classdcel_1_1_pack_d_c_e_l}.
\end{DoxySeeAlso}
The "{}\+DCEL"{} is a common way that computer scientists encode graphs; it\textquotesingle{}s also called a \textquotesingle{}half-\/edge\textquotesingle{} structure. I have converted the combinatorics underlying \doxyref{Circle\+Pack}{p.}{classall_mains_1_1_circle_pack} to use this model (at the suggestion of John Bowers).

\textquotesingle{}\doxyref{Circle\+Pack}{p.}{classall_mains_1_1_circle_pack}\textquotesingle{} are marked with "{}\+NEEDED FOR CIRCLEPACK"{}.

Note on indices:\+ Vertices, edges, and faces all get indices; these are independent and all start indexing with 1. Those for vertices are most important, others are more ephemeral and for convenience.

The principal route to creating DCEL structures uses \textquotesingle{}bouquet\textquotesingle{}s in \textquotesingle{}Comb\+DCEL.\+d\+\_\+red\+Chain\+Builder\textquotesingle{}, which first calls \textquotesingle{}Comb\+DCEL.\+create\+VE\textquotesingle{} to build vertices and faces and set \textquotesingle{}alpha\textquotesingle{}, then creates the redchain, and then calls \textquotesingle{}\doxyref{Comb\+DCEL.\+fill\+Inside}{p.}{classdcel_1_1_comb_d_c_e_l_af09405ad2f5418f8d1896eca20575f02}\textquotesingle{} to finish. (This is done because many manipulations, such as adding barycenters, need only call \textquotesingle{}fill\+Inside\textquotesingle{} to adjust the DCEL structure if the redchain was not disturbed.) Note that these routines create only a \textquotesingle{}\doxyref{Pack\+DCEL}{p.}{classdcel_1_1_pack_d_c_e_l}\textquotesingle{} object; a call to \textquotesingle{}Data\+DCEL.\+dcel\+\_\+to\+\_\+packing\textquotesingle{} will create an associated packing.

Use \textquotesingle{}hlink\textquotesingle{} for \textquotesingle{}order\textquotesingle{} (default to \textquotesingle{}layout\+Order\textquotesingle{}) to compute centers. If \textquotesingle{}fix\+First\textquotesingle{} is false, then the first face (generally \textquotesingle{}alpha\textquotesingle{}) is already laid out, so layout remaining in order. If \textquotesingle{}fix\+First\textquotesingle{} is true, then put first edge in normalized position (namely, from 0 to point on positive y-\/axis). If \textquotesingle{}hlink\textquotesingle{} is null, use \textquotesingle{}layout\+Order\textquotesingle{}, and compute successive centers as averages using all opposite edges which are already laid out. If \textquotesingle{}hlink\textquotesingle{} is not null, successive center are computed only using the one face from \textquotesingle{}hlink\textquotesingle{}. Layout can be based on radii or (in tangency setting) using schwarzians. Side-\/pairing maps are updated.
