\doxysubsection{Detailed Description}
This class holds the structures for finite subdivision rules,. 

This class holds the structures for finite subdivision rules, which describe how a finite collection of polygonal tile types are to be subdivided so each is a finite union of tiles from this same collection.

This code is used with \textquotesingle{}\doxyref{Conformal\+Tiling}{p.}{classftn_theory_1_1_conformal_tiling}\textquotesingle{} extender, \textquotesingle{}\doxyref{Tile\+Data}{p.}{classtiling_1_1_tile_data}\textquotesingle{}, and \textquotesingle{}\doxyref{Tile}{p.}{classtiling_1_1_tile}\textquotesingle{} classes. The initial structure was adopted from Bill Floyd\textquotesingle{}s software, built in conjunction with Jim Cannon and Walter Parry. In particular, we can read their $\ast$.r "{}rules"{} files (we also augment them with some optional data or data to resolve ambiguities in general situations --- as with self-\/pasting; see below).

The \textquotesingle{}tile\+Type\textquotesingle{}s are numbers. In "{}rules"{} files they are contiguous integers starting at 4 (since 0,1,2,3 were used for rectangle edges) for labeling the rules. Currently the tile types are in one-\/to-\/one correspondence with the rules. Traditionally, each rule applies to n-\/gons for some n, though more than one rule can apply for the same n.

FUTURE:\+ We will also (eventually) allow generic subdivision rules, rules which apply to n-\/gons for any n (e.\+g., the tiling version of \textquotesingle{}pentagonal\textquotesingle{} subdivision). We also wish to include some more general tiles:\+ e.\+g. uni-\/gons, bi-\/gons, multiply connected (so edges or vertices may be identified), or with penetrating edges, etc.

For each \textquotesingle{}\doxyref{Tile}{p.}{classtiling_1_1_tile}\textquotesingle{}, the first entry \textquotesingle{}vert[0]\textquotesingle{} in its vertex list is treated as its "{}principal vertex"{} (the beginning of the 0th edge the way CFP conceived of things).

OPTIONAL INFO:\+ I\textquotesingle{}ve added four optional data sections which must come at the end of the rules file:\+
\begin{DoxyItemize}
\item Edge\+\_\+marks:\+ \{t\} \{i\} \{m\} (tile type, edge index, and \textquotesingle{}mark\textquotesingle{})
\item Tile\+\_\+marks:\+ \{t\} \{n\} \{m\} (tile type, index, and \textquotesingle{}mark\textquotesingle{}) ... (done)
\item Tile\+\_\+flowers:\+ (not yet fully developed, look in \textquotesingle{}read\+Rules\+File\textquotesingle{})
\item Euclidean\+\_\+location\+\_\+data Type\+\_\+number\+\_\+(the\+\_\+first\+\_\+one\+\_\+should\+\_\+be\+\_\+4\+\_\+since\+\_\+the\+\_\+ends\+\_\+are\+\_\+typed\+\_\+0\+\_\+1\+\_\+2\+\_\+3):\+ t Eucl\+\_\+corner (standard position, first corner at 0, next at 1 0.\+0 0.\+0 1.\+0 0.\+0 x3 y3 .... Subtile\+\_\+base\+\_\+endpoints\+\_\+(z.\+x\+\_\+z.\+y\+\_\+w.\+x\+\_\+w.\+y):\+ x0 y0 x1 y1 ...

\textbf{ Subdivision\+Rules} tiling.\+Subdivision\+Rules.\+manipulate\+Rules (\begin{DoxyParamCaption}\item[{Buffered\+Reader}]{fp}{, }\item[{String}]{cmd}{, }\item[{String}]{newname}{}\end{DoxyParamCaption})}



This manipulates a rules file and saves it into given directory/\+name. It then reads the new file in the usual way to continue. Note that optional data (like euclidean corners, etc.) is not copied into the new rules file. The manipulations are very limited for now and can be done only one at a time. 


\begin{DoxyItemize}
\item given "{}-\/r s t"{}, rotate subtile s (indexed from 0, as in Floyd\textquotesingle{}s rules file) within the tile of type t by one edge counterclockwise.
\item given "{}-\/d t j i"{}, we divide tile type t into two tiles, one replaces type t and has the same number of edges and vertex j replaced by j\textquotesingle{} (order of vertices preserved), while the other is a quad with corners (j,j+1,j\textquotesingle{},j-\/1) with the i\+\_\+th one being the designated corner.
\end{DoxyItemize}

Unfortunately, we can do minimal consistency at this time. The result is saved with the given name in the given directory (defaulting to CPFile\+Manager.\+Script\+Directory.

If we successfully write the new file, we the call \textquotesingle{}read\+Rules\+File\textquotesingle{} on it and continue.
