\doxysubsection{Detailed Description}
A "{}tile"{} is a combinatorial polygon with \textquotesingle{}vert\+Count\textquotesingle{} vertices; the. 

A "{}tile"{} is a combinatorial polygon with \textquotesingle{}vert\+Count\textquotesingle{} vertices; the class extends \textquotesingle{}Face\textquotesingle{}, so has additional information, such as color, tile type, mark, etc. \textquotesingle{}vert\textquotesingle{} is the counterclockwise oriented list of corner vertices; the first, \textquotesingle{}vert[0]\textquotesingle{} is known as the \textquotesingle{}principal\textquotesingle{} vertex.

The vertex list may have been extended to support drawing operations, hence the \textquotesingle{}aug\+Vert\textquotesingle{} list:\+ this may be null, otherwise is used to avoid combinatorial ambiguities in tracing the border after refinements. Vertices from subsequent \textquotesingle{}hex\+\_\+refine\textquotesingle{} operations are NOT added to these lists, since they will be followed as hex-\/extended edges.

Each tile gets a \textquotesingle{}bary\+Vert\textquotesingle{}, the barycenter of the combinatorial face. Note, however, that this is not saved when writing tilings to files, so the information may not be set correctly when a tiling is read in for an existing packing. (Consider using \textquotesingle{}pave\textquotesingle{} command.)

A tile is identified by \textquotesingle{}tile\+Index\textquotesingle{} (starting with 1). \textquotesingle{}vert\textquotesingle{} is list of its vertices (generally legitimate circle indices in a packing P), overriding the old limit of three vertices in \textquotesingle{}\doxyref{Face.\+vert}{p.}{classcombinatorics_1_1komplex_1_1_face_a45c1c8f89779cd8912af2f411aeaf853}[]\textquotesingle{}. Its tile neighbors in ccw order are listed in \textquotesingle{}tile\+Flower\textquotesingle{}; jth entry is index of tile across edge (vert[j],vert[j+1]) (though vert[j] and vert[j+1] may not be neighbors in P any longer). If no tile across that edge, entry is 0. Note:\+ same neighbor may be listed more than once if tiles share more than one edge, perhaps even non-\/contiguous (which may cause problems).

As the associated \textquotesingle{}dual\textquotesingle{} and \textquotesingle{}quad\textquotesingle{} tilings are generated, we build white/\+grey faces, a pair for each edge. The white/\+grey (\textquotesingle{}\doxyref{Tile\+Data.\+wg\+Tiles}{p.}{classtiling_1_1_tile_data_a63c9605662b4ec93835d4c58fde29a79}\textquotesingle{}) are associated (a la dessins) with triples \{0,1,infty\} of vertices (0), midpoints of original polygon edges (1), and bary\+Vert (infty).

There may be several stages of tilings, typically from multiple stages of subdivision. The \textquotesingle{}\doxyref{Tile\+Data}{p.}{classtiling_1_1_tile_data}\textquotesingle{} to which this tile belongs is saved and if this tile has been subdivided, pointer \textquotesingle{}my\+Tile\+Data\textquotesingle{} gives its \textquotesingle{}\doxyref{Tile\+Data}{p.}{classtiling_1_1_tile_data}\textquotesingle{} object.

\textbf{ Pack\+Data} tiling.\+Tile.\+single\+Canonical (\begin{DoxyParamCaption}\item[{int}]{mode}{}\end{DoxyParamCaption})}



Create the canonical packing for this single tile, almost always in mode=3. However, there are three modes:\+ simple mode, mode=1, provides an n-\/flower for an n-\/sided tile; mode=2 adds a barycenter to each edge; mode=3 barycentrically subdivides each triangle from mode 2. 

In mode 3, vertex \textquotesingle{}mark\textquotesingle{}s are set to \{1,2,3\}:\+ 1 = tile bary\+Vert (value infty); 2 = original tile corner (value 0); 3 = edge barycenter (value 1). Hex centers of barycentrically subdivided faces are marked -\/1. We attach corresponding \textquotesingle{}\doxyref{Tile\+Data}{p.}{classtiling_1_1_tile_data}\textquotesingle{}.

The \textquotesingle{}p.\+vertex\+Map\textquotesingle{} is non-\/standard:\+ \textquotesingle{}vert\textquotesingle{} entries are changed, and $<$tindx,v$>$ in \textquotesingle{}vertex\+Map\textquotesingle{} shows that the tile corner \textquotesingle{}vert[tindx]\textquotesingle{} is now represented by the circle \textquotesingle{}v\textquotesingle{} of the packing.

We do NOT make any edge or vertex self-\/identifications, that will be done elsewhere, if needed. \doxyref{Vertex}{p.}{classcombinatorics_1_1komplex_1_1_vertex} indices are distinct.
