\doxysubsection{Detailed Description}
The \textquotesingle{}variables\textquotesingle{} package is for handling variables in. 

The \textquotesingle{}variables\textquotesingle{} package is for handling variables in \textquotesingle{}\doxyref{Circle\+Pack}{p.}{classall_mains_1_1_circle_pack}\textquotesingle{}. Basically, any string can be treated as a named variable and associated with a string \textquotesingle{}value\textquotesingle{}. Initially, definitions are limited (e.\+g., \textquotesingle{}X:\+= 4.\+5\textquotesingle{}), but additional ways are anticipated, as via queries, \textquotesingle{}r:\+=?rad 12\textquotesingle{}.

On executing commands, variable name \textquotesingle{}X\textquotesingle{} is specified via \textquotesingle{}\+\_\+X \textquotesingle{} (note the trailing whitespace to delineate the variable name string) and is simply replaced by its current value string (which is then processed in the usual way, so it may be a number, a string, etc.)

I am splitting off the GUI components, which allow for input/\+display and for slider control. For example, the keyword "{}[\+SLIDER..] "{} at beginning of value string means the variable is under slider control, see \textquotesingle{}\doxyref{Slider\+Control\+Panel}{p.}{classvariables_1_1_slider_control_panel}\textquotesingle{}. \doxyref{Circle\+Pack}{p.}{classall_mains_1_1_circle_pack} will then look for or adjust the value via the \doxyref{Slider\+Panel}{p.}{classvariables_1_1_slider_panel} in Variables tab of the \textquotesingle{}Pack Info\textquotesingle{} frame.

Only restriction on names/\+values:\+ they cannot contain \textquotesingle{};\textquotesingle{}.
