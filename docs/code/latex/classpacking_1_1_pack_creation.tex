This builds several generations of packing for the famous Kagome lattice, popular in theoretical physics and material science. For us, this is a tiling, a pattern of hexagons and triangles. We build the tile data by hand and then get the packing. 

To index vertices forming the tiles, we use the regular hex grid, e.\+g., identified with span of independent vectors u=$<$1/\+2,-\/sqrt(3)/\+2$>$ and w=$<$1/\+2,sqrt(3)/\+2$>$, so $<$i,j$>$ is interpreted as i$\ast$u+j$\ast$w. We start with a fundamental domain consisting of a hex tile and two attached triangle tiles which fill the "{}square"{} with corners (-\/1,-\/1), (1,-\/1), (1,1), (-\/1,1) We can then shift it by (2i,2j) for i=0,1,...,n and j=0,1,....,n to fill out a large "{}square"{} in u,v.

Create \textquotesingle{}maxgens\textquotesingle{} generations of the Cayley graph of the triangle group \{A/\+2, B/\+2, C/\+2\}. A, B, C are the degrees of the vertices, which are marked as 1, 2, 3, resp. One and only one may be zero, indicating a type of j-\/function with logarithmic branching. 

Typical classical cases:\+ A=2, B=3, C=5:\+ Schwarz triangles tiling the sphere A=0, B=2, C=3:\+ the classical j-\/function in hyp plane A=B=C=7:\+ constant 7-\/degree packing in hyp plane

Geometry is determined by r=2$\ast$(1/\+\+A+1/\+\+B+1/\+C). if r$>$1 sph; r=1 eucl; and r$<$1 hyp. If one of A,B,C is odd, then the others must be equal.\+The calling routine has checked validity of parameters. We compute/\+set radii using trig, but the calling routine repacks. We set face colors, blue for oriented, red for reverse oriented.

Our method is to continually expand a bdry chain of \textquotesingle{}growbdry\textquotesingle{} objects by proceeding cclw and completing flowers as we go. Each completed flow adds to growing \textquotesingle{}bouvec\textquotesingle{} vector that will lead to the final \textquotesingle{}bouquet\textquotesingle{}.

Implementation of Scott Scheffield\textquotesingle{}s \textquotesingle{}necklace\textquotesingle{} construction for random planar triangulations (at least as described by Gill and Rohde). 

Unfortunately, these triangulations will often have multiple edges. Therefore, we make each face a hex flower, i.\+e., a barycentrically subdivided face, so we get legal triangulations at each step.

Each new hex face has 1 as barycenter 2,4,6 as vertices, and 3, 5, 7 as edge barycenters. (These, of course, are renumbered as the construction goes on.) 7 will play a special role. ~\newline


Reinterpreted:\+ the "{}active"{} edge is always the edge which contained the last "{}7"{} edge barycenter. In the Rohde/\+\+Gill terminology, "{}2"{} is the "{}blue"{} end, "{}6"{} is the "{}red"{} end.

At each stage we\textquotesingle{}ll add a new face based on two coin flips:\+ rb\+Coin:\+ true = reset \textquotesingle{}red\textquotesingle{} end; false = reset \textquotesingle{}blue\textquotesingle{} end. new\+Coin:\+ true = add new vertex; false = connect existing vertices Relative to original terminology, rb\+Coin corresponds to \textquotesingle{}red\textquotesingle{} or \textquotesingle{}blue\textquotesingle{} action (r/\+R or b/\+B); new\+Coin indicates whether to use the capital or lower case symbol.

Also must maintain \textquotesingle{}blue\+Vert\textquotesingle{} and \textquotesingle{}red\+Vert\textquotesingle{} designation, our version of the integers on the negative and positive, resp., real axis in the original terminology.

We keep track of these things:\+
\begin{DoxyItemize}
\item Linked list giving the oriented boundary.
\item First in bdry list is always the current \textquotesingle{}S\textquotesingle{}. This corresponds to the "{}7"{} of the last appended face.
\item \textquotesingle{}red\+Vert\textquotesingle{} and \textquotesingle{}blue\+Vert\textquotesingle{}
\item list of face center vertices -- associate with face number
\end{DoxyItemize}

We have only few situations, indicated by \{i,j,k\}, for the adjoin operation of p1 (the growing complex) and p2 (the hex face):\+
\begin{DoxyItemize}
\item i:\+ go i steps from current S (i=1 or 3)
\item j:\+ attach j vertex of new hex face (j=2 or 4)
\item k:\+ attach k edges (k=2 or 4) Also, when new\+Coin is false (hence adding to an \textquotesingle{}old\textquotesingle{} vertex),
\item if S-\/1 = \textquotesingle{}red\+Vert\textquotesingle{} and coin flip specifies red, then \textquotesingle{}red\+Vert\textquotesingle{} is set to new "{}6"{}.
\item if S+1 = \textquotesingle{}blue\+Vert\textquotesingle{} and coin flip specifies blue, then \textquotesingle{}blue\+Vert\textquotesingle{} is set to new "{}2"{}.
\end{DoxyItemize}

Outline:\+ Assume flips give R/\+B (red/\+blue) and N/\+O (new/\+old). For R-\/O or B-\/O, have to check S-\/1 (clockwise \textquotesingle{}red\textquotesingle{}, end of bdry list) or S+1 (counterclockwise \textquotesingle{}blue\textquotesingle{}) to see if they are the \textquotesingle{}red\+Vert\textquotesingle{}, \textquotesingle{}blue\+Vert\textquotesingle{}, respectively. In this case, instead treat as though vert is N (new) and we add new vertex to the list.
