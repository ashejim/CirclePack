\doxysubsection{Detailed Description}
Abstract base for circle-\/packing repacking algorithms (hyperbolic, Euclidean, spherical). 

A "{}repack"{} iteratively adjusts circle radii until the angle sum at every interior vertex matches its target aim. This is the core numerical engine of \doxyref{Circle\+Pack}{p.}{classall_mains_1_1_circle_pack}. Subclasses implement geometry-\/specific angle-\/sum and radius-\/adjustment formulas:\+
\begin{DoxyItemize}
\item {\ttfamily \doxyref{Eucl\+Packer}{p.}{classre_pack_1_1_eucl_packer}} — Euclidean geometry.
\item {\ttfamily \doxyref{Hyp\+Packer}{p.}{classre_pack_1_1_hyp_packer}} — Poincaré disc (hyperbolic) geometry.
\item {\ttfamily \doxyref{GOpacker}{p.}{classre_pack_1_1_g_opacker}} — linearized Euclidean packer (Collins–\+Orick–\+Stephenson).
\item {\ttfamily \doxyref{Sph\+Packer}{p.}{classre_pack_1_1_sph_packer}} — spherical geometry (stub).
\end{DoxyItemize}

By design, initial data may be cloned so that the computation can run in a separate thread, be interrupted and restarted, and original data can be retained in case of failure.

Ken Stephenson
\end{DoxyAuthor}

When this is available, we will need criteria/\+options for using it in place of current java code.
