\doxysubsection{Detailed Description}
Euclidean circle-\/packing repacker using DCEL data structures. 

Implements Thurston\textquotesingle{}s iterative radius-\/adjustment algorithm for Euclidean geometry. The core loop ({\ttfamily \doxyref{continue\+Riffle}{p.}{classre_pack_1_1_eucl_packer_a8bfd65ca2291ddc0f43bb5e5bfc0ae5e}}) works as follows:\+


\begin{DoxyEnumerate}
\item For each adjustable vertex v, compute the current angle sum θ(v) from the surrounding radii.
\item Use the "{}uniform neighbor model"{} (UNM) to predict a new radius:\+ if v has 2N neighbors and the target angle sum is α, then r\+\_\+new = r · sin(θ/\+2N) · (1 − sin(α/\+2N)) /\+ [sin(α/\+2N) · (1 − sin(θ/\+2N))].
\item Apply a "{}superstep"{} acceleration:\+ extrapolate along the direction of change by a factor λ, computed adaptively to ensure the accumulated error decreases.
\item Repeat until the L² error ‖θ − α‖₂ drops below tolerance or the pass limit is reached.
\end{DoxyEnumerate}

Also provides {\ttfamily \doxyref{d\+\_\+old\+Reliable}{p.}{classre_pack_1_1_eucl_packer_af28704bd3fbaded4da3fff1531a53978}}, a simpler but more robust fallback that uses a secant-\/method radius calculation per vertex, accommodating general inversive distances.

Continue the iterative repacking loop with superstep acceleration. 

This is the main computational engine. Each pass consists of:\+
\begin{DoxyEnumerate}
\item A UNM (uniform neighbor model) radius update for every adjustable vertex.
\item A superstep:\+ extrapolate the radius changes by factor λ to accelerate convergence, subject to safety bounds.
\item A verification pass:\+ recompute angle sums with extrapolated radii and check that the error actually decreased.
\end{DoxyEnumerate}

The superstep uses two strategies (key=1 and key=2) and switches between them adaptively based on the ratio of actual vs. predicted improvement.

"{}\+Old reliable"{} repacking:\+ simple iterative secant method per vertex. 

This is the fallback algorithm that accommodates inversive distances (non-\/tangency packings) where the UNM-\/based superstep methods may fail. For each adjustable vertex whose angle-\/sum error exceeds a cutoff, it calls {\ttfamily \doxyref{e\+\_\+\+Rad\+Calc}{p.}{classre_pack_1_1_eucl_packer_a047da2c8b25e19114dc1f201d9d83c47}} to find a better radius via a few iterations of the secant method.

The cutoff starts at (1/\+3) × (average absolute error) and shrinks as the radii converge.

The angle-\/sum function θ(r) is monotonically decreasing in r for Euclidean geometry, which guarantees the secant method converges.

Static up/\+down (or down/\+up) Perron version of old\+Reliable. Not intended for speed, rather to try to ensure monotone behavior of radii. We use the uniform neighbor model (which theoretically avoids overshooting); this also allows us to easily convert computation to euclidean. 

\textquotesingle{}direction\textquotesingle{}:\+ radii go down only, down/\+up in turn, up only, or up/\+down in turn.

E.\+g., up/\+down means increase radii (of adjustable circles) until anglesum $<$= aim. Then subsequent downward adjustments will, in theory, producing a packing label (at adjustable circles). This \textquotesingle{}up/\+down\textquotesingle{} would be considered our standard "{}downward"{} Perron:\+ the up stage leads to a superpacking, then the downward Perron. CAUTION:\+ may not have enough passes to reach superpacking, then packing.

Return double[3]:\+ s=anglesum, a=aim [0]:\+ count of iterations, -\/1 on FAILURE [1]:\+ l$^\wedge$2 deficiency on upward, sqrt(sum (s-\/a)$^\wedge$2) when (s-\/a)$>$0 [2]:\+ l$^\wedge$2 deficiency on downward, sqrt(sum (s-\/a)$^\wedge$2) when (s-\/a)$<$0 [3]:\+ total l$^\wedge$2 error:\+ sqrt(sum(s-\/a)$^\wedge$2)

Note:\+ This code could serve as base for an arbitrary precision version in future.

Compute radii of packing to meet angle\+\_\+sum targets, specified as "{}aims"{}. A negative "{}aim"{} means that that circle should not have its radius adjusted. aim of zero is possible only for boundary circles in hyp setting.

All routines are iterative, inspired by routine suggested by Thurston. In hyperbolic case, we use \textquotesingle{}x-\/radii\textquotesingle{}, see description elsewhere. If h is infinite, store eucl radius with negative value for use in graphing its horocycle.

The \textquotesingle{}uniform\textquotesingle{} model (developed by Collins and Stephenson) computes the radius to get angle sum \textquotesingle{}aim\textquotesingle{} under the assumption that all neighbors have same radius. In eucl case, given angle sum \textquotesingle{}asum\textquotesingle{} and number of faces \textquotesingle{}num\textquotesingle{}, one can instead find the factor by which to multiply the current radius. This fits with a face-\/by-\/face computation, as in affine packing cases, where radii are attached to faces and same vertex may have several associated radii. (coded 3/\+2021).
