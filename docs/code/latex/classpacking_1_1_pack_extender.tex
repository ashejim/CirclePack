\doxysubsection{Detailed Description}
Abstract base for optional "{}extender"{} modules that add specialized commands and data to a circle packing. 

A \doxyref{Pack\+Extender}{p.}{classpacking_1_1_pack_extender} is a plug-\/in mechanism that attaches to a \doxyref{Pack\+Data}{p.}{classpacking_1_1_pack_data} and provides additional commands, data storage, and GUI elements for a specific mathematical experiment or application (e.\+g. \doxyref{Schwarzian}{p.}{classschwarz_work_1_1_schwarzian} derivatives, conformal tiling, Beltrami flows). Extenders are loaded dynamically and register their commands with the packing\textquotesingle{}s command parser.

An abstract class for optional structures/\+methods extending \textquotesingle{}\doxyref{Circle\+Pack}{p.}{classall_mains_1_1_circle_pack}\textquotesingle{}. Examples are (or will be):\+ Brooks packings, dessin d\textquotesingle{}enfants, complex analysis, curvature flow, etc. Some are called by abbreviations:\+ several are hard coded in \textquotesingle{}\doxyref{Command\+Str\+Parser.\+jexecute}{p.}{classinput_1_1_command_str_parser_ad2c354ce742f567dba1b7cd963fb0323}\textquotesingle{}, but \textquotesingle{}Pack\+Extender\+Loader\textquotesingle{} should allow the user to load self-\/written \textquotesingle{}Pack\+Extenders\textquotesingle{} (once they have been compiled to $\ast$.class)

The idea is that when a packing is created for these special purposes, then the packing points to the \doxyref{Pack\+Extender}{p.}{classpacking_1_1_pack_extender} and vice verse. Commands can be constructed that apply to a packing iff it has the appropriate extender abbreviation. \begin{DoxyAuthor}{Author}
