47 public void hyperlinkUpdate(final HyperlinkEvent e) {
48 // Old versions of Java don't support getting the time of a hyperlink event from
49 // the object itself, so use the time that this function was called instead.
50 final long eventTime = new Date().getTime();
51
52 /*
53 * Dispose of the previous frame and note that the current hyperlink event has changed.
54 *
55 * Clearly, if a hyperlink has been exited (moused out of), we would like to dipose
56 * the tooltip. If a hyperlink has been activated (clicked), we would also like
57 * to dispose the tooltip. If a hyperlink has been entered (moused over), the
58 * tooltip should already be disposed from a previous exit or never displayed. The
59 * redundancy doesn't hurt and is faster than checking.
60 *
61 * We track the current hyperlink event to handle thread timing issues. Later, if
62 * the user mouses over an XMD script hyperlink, we'll spin a thread to parse out
63 * the preview image, which may take a long time over the net. Once it has the
64 * preview image, it will post code to the Swing EventQueue to display it in a
65 * tooltip near the mouse. However, the user may have moused out of the hyperlink
66 * and be across the screen at this point.
67 *
68 * Thus, we compare the hyperlink event we are responding to and this instance's
69 * most current received hyperlink event. If they are not the same, we cancel the
70 * tooltip display. This behavior occurs below, in the thread definition.
71 */
72 frame.dispose();
73 currentHyperlinkEvent = e;
