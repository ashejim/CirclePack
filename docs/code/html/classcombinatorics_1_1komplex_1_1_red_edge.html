Detailed Description

Red-chain edge: tracks boundary and side-pairing structure in the DCEL.

After processing a dcel (see 'CombDCEL.redcookie'), there is (except in the case of a topological sphere) a 'redChain', a closed linked list of 'RedEdge's surrounding a simply connected patch of interior faces. In non-simply connected cases, segments of red edges may be identified. Outside the redChain are one or more ideal faces. NOTE: the dcel structure still lies with the 'HalfEdge's via 'myEdge'. This is not persistent data; these edges change, e.g., when we change the DCEL. (?? 6/2025 ??) TODO: do we need to save data when changing the DCEL?
