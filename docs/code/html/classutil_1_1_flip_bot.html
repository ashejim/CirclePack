◆ chooseMove()

int util.FlipBot.chooseMove ( int home,
int previous,
EdgeSimple newEdge )

This chooses the vertex the bot is to move to based on 'moveStrategy'; the calling routine carries out the move. If 'edge' is not null, it is the new edge encoded as (v_left, v_right) relative to the directed edge that was flipped. (Usually, if homeVert was one end of old edge, this is (v_left, v_right) as seen from the 'homeVert'.).

The combinatorics from the flip should already have been adjusted.

Parameters
inthome, home vertex (before flip)
intprevious, previous vertex (before flip)
newEdgeEdgeSimple, new edge (v_left,v_right) or null
Returns
int, 0 if there is to be no move, -1 on error, else return new 'homeVert', 0 if no move.
